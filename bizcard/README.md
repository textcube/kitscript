# BizCard Pro — QR Business Card Designer

브라우저에서 QR 코드가 포함된 디지털 명함을 실시간으로 디자인하고 PNG로 내보내는 단일 파일(`index.html`) 웹 앱.

## 현재 구조 분석

### 기술 스택
- **단일 HTML 파일** (558줄) — 빌드 도구 없음, 정적 호스팅 가능
- **외부 의존성 (CDN)**: qrcode-generator 1.4.4, html2canvas 1.4.1, Google Fonts
  - 원래 QRCode.js 1.0.0을 사용했으나, 한글 등 멀티바이트 문자가 포함된 vCard에서 "code length overflow" 예외를 던지는 라이브러리 버그가 있어 UTF-8을 올바르게 처리하는 qrcode-generator로 교체함 (2차 개선에서 캔버스 직접 렌더링 방식으로 전환, quiet zone 2모듈 포함)
- **상태 관리**: 전역 `state` 객체 + 입력 이벤트마다 전체 `render()` 재실행 (QR 코드도 매번 재생성)

### 레이아웃
- 3단 그리드: 좌측 정보 입력 패널(340px) / 중앙 미리보기 / 우측 디자인 컨트롤 패널(320px)
- 850px 이하에서 1열로 세로 스택

### 주요 기능
| 기능 | 설명 |
|------|------|
| 정보 입력 | 이름, 직함, 회사, 전화, 이메일, 웹사이트, YouTube, 주소 (빈 필드는 자동 숨김) |
| 에셋 업로드 | 회사 로고, 프로필 사진 (FileReader → dataURL) |
| QR 모드 | vCard(연락처 추가) / 웹사이트 링크 / YouTube 링크 |
| 템플릿 | Corporate / Minimal / Dark (3종) |
| 커스터마이즈 | 배경색·그라디언트, 텍스트 색, 폰트 4종, 크기 스케일, QR 색상, QR 중앙 로고 오버레이 |
| 내보내기 | html2canvas로 명함 전체 PNG(3배 해상도), QR 단독 PNG |

## 발견된 문제점

1. **모바일 UX 미흡** — 850px 이하에서 단순 세로 스택이라 미리보기가 입력 폼 아래로 밀려 실시간 확인이 불가능. 명함이 500px 고정 폭이라 375px 화면에서 잘림. 헤더의 내보내기 버튼 2개도 좁은 화면에서 넘침.
2. **밋밋한 흰색 위주 UI** — 앱 배경(#f1f5f9)과 패널(#ffffff)이 전형적인 관리자 툴 느낌. 고급스러운 인상이 없음.
3. **템플릿 중복** — Corporate와 Minimal이 색상 설정이 완전히 동일(레이아웃만 다름). 템플릿 수 자체도 3종으로 부족.
4. **잔버그**
   - `generateQRData()`에 `qrMode === 'email'` 분기가 있으나 select에 해당 옵션이 없음 (죽은 코드)
   - `initQR()`은 QR 크기 110, `render()`는 105로 불일치 (initQR 자체가 사실상 불필요 — 직후 render가 재생성)
   - vCard 줄바꿈이 `\n`인데 스펙(RFC 6350)상 `\r\n`이어야 일부 스캐너에서 안정적
   - vCard 필드에 `;`, `,`, `\n` 등 특수문자 이스케이프 없음
5. **연락처 아이콘이 이모지(📞✉️🌐)** — OS마다 렌더링이 다르고 html2canvas 내보내기 시 깨지거나 톤이 안 맞음. 고급스러운 명함과 부조화.
6. **편의 기능 부재** — 입력 내용 localStorage 저장 없음(새로고침 시 소실), 명함 뒷면 없음, 실행 취소 없음.
7. **접근성** — 템플릿 선택이 `div onclick`(키보드 접근 불가), 색상 입력에 연결 label 미흡.

## 개선 지시 사항 (→ Sonnet 5 작업)

> 아래 항목은 Sonnet 5 에이전트가 `index.html`을 직접 수정하여 구현한다. 단일 파일 구조와 기존 기능(입력→실시간 미리보기→PNG 내보내기)은 유지한다.

### 1. 프리미엄 다크 UI로 전면 리디자인 (최우선)
- 흰색 배경 탈피: 앱 전체를 딥 다크 톤(예: #0b0f1a ~ #131a2a 계열) + 은은한 그라디언트/글로우 배경으로 전환
- 패널은 글래스모피즘(반투명 + blur) 또는 어두운 카드 스타일, 골드/바이올렛 계열 액센트로 고급스러운 분위기
- 버튼·입력 필드·슬라이더·컬러피커까지 다크 테마에 맞게 일관되게 스타일링
- 미리보기 영역에 스포트라이트/무대 조명 느낌의 배경 처리로 명함이 돋보이게

### 2. 모바일 최적화 (최우선)
- 모바일(≤850px)에서 **미리보기를 상단에 sticky로 고정**하고, 그 아래 편집 영역을 탭(정보 / 디자인 / QR)으로 전환하는 구조
- 명함 미리보기는 화면 폭에 맞춰 자동 축소(transform scale 계산, 500px 고정 폭 문제 해결)
- 터치 타깃 최소 44px, 헤더 버튼은 모바일에서 아이콘화 또는 하단 고정 바로 이동
- 폰트 크기·간격 모바일 가독성 점검

### 3. 명함 템플릿 고급화
- Corporate/Minimal 색상 중복 해소, 각 템플릿에 고유한 정체성 부여
- 프리미엄 템플릿 2종 이상 추가 (예: 골드 액센트의 "Luxury", 그라디언트 글로우 "Aurora" 등)
- 이모지 연락처 아이콘을 인라인 SVG 아이콘으로 교체 (내보내기 품질·통일감 확보)

### 4. 버그 수정
- vCard 줄바꿈 `\r\n` 적용 및 특수문자(`;` `,` 개행) 이스케이프
- 죽은 코드 정리: `qrMode === 'email'` 분기 제거 또는 select에 이메일 옵션 추가 중 택일
- `initQR()`/`render()` QR 크기 불일치 해소 (중복 초기화 제거)

### 5. 편의 기능
- `state`를 localStorage에 자동 저장/복원 (업로드 이미지 포함, 용량 초과 시 이미지 제외 폴백)
- 템플릿 선택을 `button` 요소로 변경해 키보드 접근성 확보

### 검증 기준
- 375px(모바일) / 768px(태블릿) / 1280px(데스크톱)에서 레이아웃 깨짐 없음
- 명함 PNG 내보내기와 QR 스캔(vCard 연락처 추가)이 기존과 동일하게 동작
- 콘솔 에러 0건

## 2차 개선 지시 사항 (→ Sonnet 5 작업)

> 1차 작업(다크 UI, 모바일 탭, 템플릿 5종, localStorage) 완료 후 추가 지시.

### 6. 폰트 셀렉션 세련화
- 현재 4종(Inter/Montserrat/Playfair/Roboto Mono)은 평범한 인상. 고급 명함에 어울리는 폰트로 교체·확충
- 이름(헤딩)용 세리프/디스플레이 폰트와 본문용 산세리프를 **페어링 프리셋**으로 제공 (예: Cormorant Garamond + Manrope, Marcellus + Sora 등)
- 한글 명함 대응: Noto Serif KR / Noto Sans KR 등 한글 지원 폰트 포함
- 템플릿마다 어울리는 기본 폰트 페어가 자동 적용되도록

### 7. 배경 타일/패턴 선택
- 명함 배경에 은은한 패턴 오버레이 선택 UI 추가 (None / 도트 / 그리드 / 웨이브 / 지오메트릭 등 4종 이상)
- 인라인 SVG data-URI 배경으로 구현하여 html2canvas 내보내기에서 깨지지 않아야 함
- 패턴 투명도(강도) 조절 슬라이더 제공

### 8. 고해상도 QR 생성·출력
- 현재 QR은 화면 표시 크기(~105px canvas)로 생성되어 내보내기 시 저해상도 → 인쇄 부적합
- QR을 내부적으로 고해상도(≥1000px)로 생성하고 화면에는 축소 표시, "Save QR Only"는 고해상도 원본을 저장
- 명함 PNG 내보내기에서도 QR이 선명하게 나오도록 (고해상도 canvas의 dataURL을 img로 표시하는 방식 등)
- QR 단독 저장 시 해상도 선택 옵션(예: 512 / 1024 / 2048px) 고려

> **2차 작업 완료 노트**: 위 3개 항목 모두 구현 완료. 검증 중 한글 이름이 포함된 vCard에서 QRCode.js 1.0.0의 "code length overflow" 버그가 발견되어 qrcode-generator 1.4.4로 교체하고 캔버스 직접 렌더링으로 전환함. jsqr 디코딩 테스트로 한글 vCard가 정확히 스캔되는 것을 확인.

## 3차 개선 지시 사항 (→ Sonnet 5 작업)

### 9. 자동 시연 모드 (게임의 어트랙트 모드)
- **발동 조건**
  - 최초 실행 시 (localStorage에 저장된 상태가 없을 때): 로드 후 잠시 뒤 자동 시작
  - 일정 시간(예: 60초) 아무 입력이 없고, 사용자가 이번 세션에서 수정한 내용이 없을 때
- **시연 내용**: 샘플 인물(한글/영문 페르소나)로 템플릿 5종을 순회하며 각 템플릿에 어울리는 폰트 페어·패턴·QR 스타일을 함께 전환. 부드러운 전환 간격(예: 3~4초)
- **종료 조건**: 클릭/터치/키 입력 등 어떤 상호작용이든 즉시 종료하고 시연 전 사용자 상태로 완전 복원
- **안전 장치**
  - 시연 시작 전 상태 스냅샷 → 종료 시 복원, 시연 중에는 localStorage 자동 저장 중단 (시연 데이터가 사용자 데이터를 덮어쓰면 안 됨)
  - 시연 중임을 알리는 배지 표시 ("Demo — 클릭하면 종료" 등)
  - 시연 종료 후에는 다시 유휴 타이머 재시작

## 4차 개선 지시 사항 (→ Sonnet 5 작업)

> 컨셉: **"움직이는 명함"**. 로고·텍스트·QR에 트위닝 기반 모션을 입히고, 그 결과물을 영상(WebM)으로 저장하거나 의존성 없는 독립 HTML 페이지로 bake한다. 단일 파일 구조와 기존 기능(PNG/QR 내보내기, localStorage, 데모 모드)은 그대로 유지한다.

### 10. 모션 프리셋 시스템 (트위닝)

- `state`에 `motion`(프리셋 id), `motionSpeed`(0.5~2.0, 기본 1.0) 필드 추가. localStorage 저장/복원 대상에 포함
- 모션은 **엔트런스(등장) + 앰비언트(상시 루프)** 2계층으로 설계:
  - 엔트런스: 요소별 스태거 — 로고 → 이름 → 직함/회사 → 연락처 행(위→아래) → QR 순으로 60~120ms 간격 등장
  - 앰비언트: 엔트런스 완료 후 은은한 무한 루프 (로고 플로팅, 광택 스윕, QR 미세 펄스 등). 명함답게 절제된 강도로
- 프리셋 5종 (None 포함):
  | 프리셋 | 엔트런스 | 앰비언트 | 어울리는 템플릿 |
  |--------|----------|----------|------------------|
  | None | 없음 (즉시 표시) | 없음 | — |
  | Fade Up | 아래→위 페이드+슬라이드 | 거의 없음 (절제) | Corporate, Minimal |
  | Slide | 좌측 슬라이드인 + 액센트 라인 스윕 | 액센트 라인 은은한 글로우 | Dark |
  | Pop | scale 오버슈트(back easing) | 로고 미세 플로팅 | Aurora |
  | Shine | 페이드인 | 로고·이름에 주기적 광택 스윕(gradient sweep) | Luxury |
- 구현은 **CSS keyframes + 클래스 토글** 기본, 필요 시 Web Animations API 병용. **트윈 라이브러리 CDN 추가 금지** (현재 CDN 2개 유지)
- `TEMPLATE_DEFS`에 템플릿별 기본 모션 추가 (위 표 기준). `applyTemp()` 시 함께 적용
- Design 패널에 "Motion" 섹션 추가: 프리셋 그리드(`temp-item` 스타일 재사용), 속도 슬라이더, ▶ Replay 버튼
- **재생 타이밍 규칙 (중요)**: 매 `render()`마다 엔트런스가 재시작되면 안 됨. 재시작 트리거는 (a) 프리셋 변경 (b) 템플릿 변경 (c) Replay 버튼 (d) 데모 스텝 전환 — 이 4가지뿐. 텍스트 타이핑 중에는 앰비언트 루프만 유지
- `prefers-reduced-motion: reduce`에서는 엔트런스를 즉시 완료 상태로, 앰비언트 루프 정지
- **데모(어트랙트) 모드 통합**: `DEMO_STEPS` 각 스텝에 어울리는 모션 프리셋을 지정하고 스텝 전환마다 엔트런스를 재생 → 시연 자체가 모션 쇼케이스가 되도록. 데모 종료 시 사용자의 모션 설정도 스냅샷에서 복원

### 11. 영상 내보내기 (WebM)

- 헤더와 모바일 액션바에 "Export Video" 버튼 추가
- 권장 구현 — **2-pass 오프라인 캡처 후 실시간 인코딩**:
  1. **캡처 패스**: 모션 타임라인을 결정론적 함수(t → 각 요소의 transform/opacity)로 정의하고, t를 스텝하며 html2canvas로 프레임 시퀀스 캡처 (예: 4초 × 20fps = 80프레임, scale 2). **캡처 중에는 CSS 애니메이션을 끄고 동일 타임라인의 인라인 스타일을 t마다 직접 적용** (getComputedStyle의 중간값 캡처에 의존하지 말 것), 캡처 후 원상 복구
  2. **인코딩 패스**: 캡처된 프레임을 실시간 속도로 canvas에 재생하면서 `canvas.captureStream()` + MediaRecorder로 WebM 인코딩 (오프라인 프레임에 realtime 타임스탬프가 찍히는 문제 회피)
- 영상 길이 = 엔트런스 1회 + 앰비언트 1~2사이클. 마지막 프레임이 앰비언트 시작 프레임과 자연스럽게 이어져 루프 재생에 적합하게
- 캡처에 수 초 걸리므로 진행률 표시(오버레이 또는 버튼 텍스트), 완료 시 `BizCard_이름.webm` 다운로드
- MediaRecorder/코덱 미지원 브라우저에서는 안내 후 버튼 비활성화 폴백. WebCodecs는 muxer 의존성이 추가로 필요하므로 채택하지 않음

### 12. 독립 HTML 페이지 Bake

- "Bake HTML" 버튼: 현재 명함을 **모션 포함, 의존성 없는 단일 HTML 파일**로 생성해 다운로드 (Blob → `BizCard_이름.html`)
- baked 페이지 구성:
  - 다크 스테이지(중앙 정렬 + 은은한 배경 글로우) 위에 명함 1장, 로드 시 엔트런스 1회 재생 후 앰비언트 무한 루프, 클릭/탭하면 엔트런스 리플레이
  - 로고·프로필은 이미 dataURL, QR은 고해상도 canvas의 dataURL로 인라인 → **JS 라이브러리(qrcode-generator, html2canvas) 포함 금지**, 정적 마크업+CSS(+리플레이용 소량 JS)만으로 동작
  - 외부 요청은 Google Fonts `<link>` 1개만 허용, 오프라인에서도 폴백 폰트로 열리게
  - `<title>`은 "이름 — 회사", 모바일에서 명함이 화면 폭에 맞게 scale 되도록 뷰포트 대응
- **XSS 필수 방어**: state의 텍스트 필드를 baked HTML에 삽입할 때 반드시 이스케이프 (shop/quiz에서 같은 계열의 baked-output XSS를 수정한 이력이 있음 — 동일 기준 적용)
- (선택) 하단에 "Save Contact" 링크: QR과 동일한 vCard를 `data:` URI 다운로드로 제공. 구현 부담이 크면 생략 가능

### 검증 기준

- 모션 프리셋 전환·속도 조절·Replay가 즉시 반영되고, **타이핑 중 엔트런스 재시작이 없어야 함**
- Export Video: Chrome에서 WebM이 다운로드되고 재생이 부드러우며(끊김·프레임 중복 없음) 길이 오차 ±10% 이내
- Bake HTML: 결과 파일을 단독으로 열었을 때 모션 포함 정상 동작, 콘솔 에러 0건, 외부 요청은 폰트뿐임을 네트워크 탭으로 확인
- 기존 PNG/QR 내보내기 회귀 없음 — **PNG는 모션의 최종 정지 상태로 캡처** (등장 중간 상태가 찍히면 안 됨)
- 데모 모드에서 스텝마다 모션이 재생되고, 종료 시 사용자 상태(모션 설정 포함)가 완전 복원됨
- 375px / 768px / 1280px에서 새 Motion 섹션·버튼 포함 레이아웃 깨짐 없음

> **4차 작업 완료 노트**: 항목 10~12 모두 구현·검증 완료 (콘솔 에러 0, 타이핑 중 엔트런스 재시작 없음, WebM 다운로드 확인, baked HTML 단독 실행·XSS 이스케이프·외부 요청 폰트 1건 확인, PNG는 최종 정지 상태 캡처, 3개 뷰포트 레이아웃 정상). 구현 중 발견된 함정: `bakeHTML()` 템플릿 문자열 안의 `</script>` 리터럴이 index.html 자신의 바깥 `<script>` 태그를 조기 종료시켜 앱 전체가 깨지는 문제 → 소스에서 `<\/script>`로 이스케이프하여 해결. 데모 종료 시에는 엔트런스를 리플레이하지 않고 `settleMotion()`으로 즉시 정착시켜 사용자 상태만 복원한다.

## 5차 개선 지시 사항 (→ Sonnet 5 작업)

> 컨셉: **"인쇄소에 실제로 넘길 수 있는 명함"**. 지금의 500×317px PNG는 화면용일 뿐, 실물 명함 제작에는 규격·해상도·재단 여백이 전부 부족하다. 실측 규격 기반 캔버스, 양면(뒷면), 300DPI + bleed 내보내기, PDF 제출본까지 인쇄 파이프라인 한 벌을 완성한다. CMYK 변환·멀티 카드 저장·팀 일괄 생성은 이번 범위가 **아니다** (RGB 제출 한계는 안내 문구로만 명시).

### 13. 실측 카드 규격

- `CARD_SPECS` 정의 + `state.cardSize` 추가 (localStorage 저장/복원 포함):
  | id | 실측 | 용도 |
  |----|------|------|
  | `kr` (기본값) | 90 × 54 mm | 한국 표준 명함 |
  | `credit` | 85.6 × 54 mm | 신용카드형 (현재 비율과 가장 유사) |
  | `us` | 88.9 × 50.8 mm | US 3.5 × 2 인치 |
- 미리보기는 폭 500px 고정을 유지하되 **높이를 규격 비율로 계산** (예: kr → 500×300). `updateCardScale()`·모션 캡처·bake에 하드코딩된 500/317을 모두 규격 기반 동적 치수로 교체
- Design 패널 상단에 규격 선택 UI (temp-item 그리드 재사용). 규격 변경은 레이아웃 재계산만 하고 색·폰트·모션 등 다른 상태는 건드리지 않음
- 구버전 저장분(cardSize 없음)은 `kr`로 마이그레이션

### 14. 뒷면(양면) 지원

- `state.backMode`: `none`(기본) / `brand` / `info-en` + 뒷면 전용 필드들
  - **brand**: 액센트 컬러 배경 위에 로고 + 회사명 + 태그라인(신규 입력 필드) 중앙 배치. 템플릿의 색·패턴 계열을 이어받아 앞면과 한 벌로 보이게
  - **info-en**: 한국 관습(앞면 한글/뒷면 영문) 대응 — 영문 이름/직함/회사/주소 대체 필드 세트로 앞면과 동일 레이아웃 렌더. 연락처·QR은 앞면 값 공유
- Info 패널에 "Back Side" 섹션 (모드 선택 + 해당 모드 필드)
- 미리보기에 **앞/뒤 전환 토글** — 카드 전체를 3D 회전(rotateY 180°)시키는 방식은 **회전 중 한쪽이 가려지거나 잘리는 문제가 있으므로 금지**. 대신 **중앙(세로 중심선)을 힌지로 한 페이지 넘김(page-turn) 연출**로 구현: 앞면이 책장 넘기듯 중앙 기준으로 넘어가며 뒷면이 드러나고, 되돌릴 때 역방향으로 넘겨 앞면 복귀. 회전 중 어떤 순간에도 카드가 컨테이너에 clip되거나 미리보기 영역 밖으로 벗어나지 않아야 함 (perspective/overflow 처리 포함). `backMode === 'none'`이면 토글 숨김
- 데모(어트랙트) 모드: 뒷면이 있는 스텝에서 페이지 넘김으로 뒷면을 보여줬다가 앞면으로 돌아오는 연출 1회 포함 (스냅샷 복원 규칙은 기존과 동일)
- 뒷면도 앞면과 동일하게 XSS 이스케이프 대상 (bake 포함)

### 15. 인쇄용 내보내기 — 300DPI PNG + PDF (bleed·재단선)

- **내보내기 모달로 UI 정리**: 헤더/모바일 액션바의 버튼 증식을 멈추고 "Export" 단일 버튼 → 모달에서 용도 선택: Screen PNG(기존 3x) / **Print PNG (300DPI)** / **Print PDF** / QR Only / Video (WebM) / Bake HTML / vCard(.vcf). 기존 내보내기 함수들은 모달로 진입점만 이동, 동작 회귀 금지
- **Print PNG (300DPI)**: 재단 크기 + 사방 3mm bleed를 300DPI로 환산한 픽셀 크기로 출력 (kr 기준 96×60mm → **1134×709px**). bleed 포함/미포함 선택 제공
  - 구현: 내보내기 전용 오프스크린 DOM 복제본을 bleed 포함 치수로 만들고 **배경·그라디언트·패턴은 bleed 영역까지 확장**, 텍스트 등 콘텐츠는 재단선 기준 원위치 유지 → html2canvas 캡처. 미리보기 DOM을 직접 변형하지 말 것
  - 텍스트·로고가 재단선에서 4mm 이내로 붙어 있으면 모달에 안전 영역 경고 표시 (차단은 하지 않음)
- **Print PDF**: jsPDF CDN 추가 허용 (이 목적에 한해 — 다른 라이브러리 추가는 계속 금지). 페이지 크기 = 재단 크기 + bleed + 재단선 여백(mm 단위 실측), 300DPI 캡처 이미지를 실측 mm로 배치, **재단선(crop marks) 4모서리 표기**, 뒷면 활성 시 2페이지(앞/뒤)
- 모달에 고정 안내 문구: "웹 도구 특성상 RGB로 출력됩니다. 인쇄소에서 CMYK 변환 시 색상이 다소 달라질 수 있습니다."
- 양면 활성 시 Screen/Print PNG도 앞·뒤 각각 저장 (파일명 `_front`/`_back`)

### 16. vCard(.vcf) 다운로드

- 내보내기 모달에 "Save Contact (.vcf)" 항목: `generateQRData()`의 vCard 문자열을 그대로 `text/vcard;charset=utf-8` Blob으로 다운로드 (`이름.vcf`)
- QR 모드와 무관하게 항상 vCard 데이터로 생성
- (선택) baked HTML 하단의 "Save Contact" 링크도 동일 데이터로 연결 — 4차에서 생략했다면 이번에 포함

### 검증 기준

- 규격 3종 전환 시 미리보기 비율이 실측과 일치하고, 템플릿 5종·모션·데모·bake·video 전부 회귀 없음 (500×317 하드코딩 잔존 금지)
- Print PNG: kr + bleed 기준 정확히 1134×709px, bleed 영역까지 배경·패턴이 채워져 있고 흰 띠 없음
- Print PDF: 페이지 실측 크기(mm)가 스펙과 일치, 재단선 표시, 양면 시 2페이지, 뷰어(브라우저/Acrobat)에서 정상 열림
- 뒷면: 모드 3종 전환·플립 토글 정상, info-en 필드가 bake/PNG/PDF에 이스케이프되어 반영
- vcf: 파일을 텍스트로 열어 CRLF·이스케이프·UTF-8 한글 정상 확인
- 내보내기 모달: 기존 7개 내보내기 경로 전부 모달 경유로 동작, 모바일(375px)에서 모달·버튼 조작 가능
- 콘솔 에러 0건

> **5차 작업 완료 노트**: 항목 13~16 모두 구현·검증 완료. Print PNG는 다운로드 파일 실측 1134×709px(bleed 미포함 1063×638), PDF는 pypdf 파싱으로 2페이지·108×72mm·재단선 확인, vcf는 바이트 검사로 CRLF·이스케이프·UTF-8 한글 확인. 앞/뒤 전환은 스펙 변경대로 **중앙 세로 힌지 페이지 넘김**(앞면만 0→−180° 회전, backface-visibility로 뒷면 노출)으로 구현 — 전환 중 카드가 미리보기 영역을 벗어나지 않음을 실측(26샘플)으로 검증. 스펙 이탈 결정: ① Print 복제본 해상도 확장은 CSS transform 대신 html2canvas scale 옵션 사용(transform 방식은 html2canvas가 자식 요소를 컬링해 연락처·QR이 누락되는 버그 실측), ② 인쇄 출력물에서는 카드 border 제거(1px 테두리가 재단선 위에 놓여 재단 후 불규칙하게 잘림 — 화면용에는 유지), ③ `img.decode()`를 800ms 타임아웃과 race(백그라운드 탭에서 영영 resolve 안 되는 문제 방어). 부수 수정: baked HTML의 Pop 앰비언트가 잘못된 요소 id를 참조해 동작하지 않던 4차 유래 버그 정정.
