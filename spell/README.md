# Spell Claw — by BuntGames

IT 용어(영단어)를 학습하는 횡스크롤 슈팅 게임. 단어가 뱀(지네) 형태의 적으로 등장하며,
모든 마디를 격추하면 머리 글자가 아이템으로 드롭된다. 글자를 수집해 상단의 목표 단어
3개를 완성하면 60초 보너스(오버드라이브) 모드가 시작된다.

- 단일 파일: `index.html` (HTML + CSS + Canvas 2D + Web Audio, 외부 의존성은 Google Fonts뿐)
- 조작: ↑/↓ 또는 W/S로 이동, 발사는 자동. 7초간 입력이 없으면 AUTO PILOT(자동 플레이) 복귀
- 부가 기능: 단어장 편집(SPELLBOOK EDITOR), 클립보드 붙여넣기로 단어 교체, 사운드 토글(Web Audio 신디사이저)

## 코드 구조 분석

| 영역 | 내용 |
|------|------|
| 상태 머신 | `gameState`: STARTING → PLAYING → BONUS → ENDED, 별도 PAUSED |
| 적 | `spawnEnemy()` — 단어를 마디(segment) 배열로 생성, 사인파 궤적, 마디당 HP 3 |
| 글자 수집 | 적 전멸 시 머리 글자 드롭 → `collectLetter()`로 목표 단어 채움 |
| 업그레이드 | 완성한 단어 수(0~3)에 따라 2연발/3방향+드론 확장 |
| 자동 조종 | `handleAutoAI()` — 필요한 글자 드롭 우선 추적, 없으면 선두 적 추적 |
| 렌더링 | `requestAnimationFrame` 루프에서 `update()` + `draw()` |

## 발견된 문제점 (버그)

1. **순회 중 배열 splice** — `bullets`, `enemies`, `drops`, `particles`, `floatingTexts` 모두
   `forEach` 순회 중 `splice`로 제거한다. 같은 프레임에 인접한 두 요소가 제거 대상이면
   뒤 요소가 건너뛰어져 판정 누락·잔상이 발생한다.
2. **프레임레이트 의존 게임 속도** — 고정 타임스텝이 없어 120Hz/144Hz 모니터에서 게임 전체가
   2배 이상 빨라진다. `bonusTimeLeft -= 1/60`도 60fps를 가정한다.
3. **게임오버 후 설정 버튼 잠김** — `settingsBtn` 핸들러의
   `if (gameState === 'ENDED' && Date.now() - restartTimerStart > 0) return;` 조건은
   ENDED 상태에서 항상 참이라 설정을 열 수 없다(의도 불명확한 조건식).
4. **총알-마디 충돌 판정 왜곡** — 마디 순회 안에서 `bullets.splice(bIdx, 1)`를 호출해
   같은 프레임의 다른 총알 인덱스가 밀린다 (1번과 동일 계열).
5. **사실상 없는 패배 조건** — `missedCount >= 100`이어야 게임오버라 긴장감이 없다.
6. **난이도 고정** — `INITIAL_ENEMY_SPEED`와 스폰율이 시간에 따라 변하지 않아 단조롭다.
7. **모바일 미지원** — 900×550 고정 컨테이너(작은 화면에서 오버플로), 터치 입력 없음,
   키보드 전용 조작.
8. **설정/기록 비영속** — 단어장·최고점수·음소거 설정이 새로고침 시 초기화된다
   (localStorage 미사용).
9. **PAUSED 상태 키 불일치** — 일시정지 중 `keydown`은 무시하지만 `keyup`은 처리해
   재개 후 키 상태가 어긋날 수 있다. 탭 전환(visibilitychange) 시 일시정지도 없다.
10. **드롭 글자의 한계** — 항상 단어의 첫 글자만 드롭되므로, 목표 단어의 중간 글자를 얻으려면
    해당 글자로 시작하는 단어가 단어장에 있어야 한다. `getNeededLetters()` 기반 스폰 보정이
    있지만 단어장에 따라 진행 불가(soft-lock)가 가능하다.

## 개선안 (우선순위)

### P1 — 정확성/안정성
- [x] 고정 타임스텝(accumulator 패턴) 도입: 모든 이동·타이머를 dt 기반으로 전환, 어느
      주사율에서도 동일 속도 보장 (Tank Edition 개편과 동일 방식)
- [x] 배열 제거를 역방향 for 루프 또는 `filter`로 교체해 순회-중-제거 버그 일괄 해결
- [x] 설정 버튼 ENDED 잠김 조건 수정 (게임오버 화면에서도 단어장 편집 가능하게)
- [x] soft-lock 방지: 필요한 글자로 시작하는 단어가 단어장에 없으면 해당 글자를 머리로 하는
      적을 합성 생성하거나, 드롭 글자를 "필요한 글자" 중에서 선택

### P2 — 모바일/반응형
- [x] 컨테이너를 반응형으로(`max-width` + aspect-ratio), 캔버스 DPR 대응
- [x] 터치 조작: 화면 드래그로 상하 이동(포인터 이벤트), 터치 시 MANUAL 전환
- [x] 상단 UI가 좁은 화면에서 겹치지 않게 미디어쿼리 정리

### P3 — 게임성
- [x] 난이도 곡선: 경과 시간·점수에 따라 적 속도/스폰율 점진 증가
- [x] 패배 조건 현실화: 미스 한도 10회 내외 + 상단 UI에 잔여 한도 시각화
- [x] 콤보 시스템: 연속 격추 시 점수 배율, HUD 표시
- [x] 보너스 모드 차별화(현재는 속도·스폰만 증가): 점수 2배, 전용 연출 등

### P4 — 품질/편의
- [x] localStorage 영속화: 단어장, 최고점수(HI-SCORE 표시 추가), 음소거 설정
- [x] `visibilitychange` 시 자동 일시정지, P 키로 수동 일시정지
- [x] 접근성: 아이콘 버튼 `aria-label`, `aria-pressed`(사운드 토글)
- [x] PAUSED 중 키 상태 초기화(`keys` 리셋)로 재개 후 입력 어긋남 방지

## 단어장 형식

한 줄에 `단어 뜻` (공백 구분, 뜻은 여러 단어 가능). 최소 3개 필요.

```
PROMPT 프롬프트
DEPLOY 배포하다
```
