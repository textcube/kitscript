# Cable Car Defense

설산 배경의 케이블 카 디펜스 캔버스 게임. 상단 케이블을 따라 이동하는 케이블카(포대)가 지상에서 우측 사다리를 향해 전진·등반하는 적을 요격하고, 적이 사다리 꼭대기에 도달하면 패배("CABLE CUT")하는 단일 파일(`index.html`) 웹 게임.

## 현재 구조 분석

### 기술 스택
- **단일 HTML 파일** (약 890줄) — 외부 의존성 전무, 정적 호스팅 가능
- **렌더링**: Canvas 2D 전체 화면(`100vw×100vh`) + 오프스크린 캔버스에 정적 배경(하늘 그라데이션, 프랙탈 설산 5레이어, 건물, 나무) 사전 렌더 + CSS 오버레이 HUD (무기 슬롯 5종, 쿨다운 오버레이)
- **오디오**: Web Audio API 자체 신스 (발사/폭발/방패 튕김/빙결/게임오버 톤), 기본 OFF
- **상태 관리**: 전역 `state` 객체 + `WEAPONS`/`ENEMY_TYPES` 상수 배열, 엔티티는 평범한 객체 배열
- **게임 루프**: `requestAnimationFrame` 직결 (update → draw), 쿨다운·스폰 주기만 `Date.now()` 기반

### 주요 기능
| 기능 | 설명 |
|------|------|
| 무기 5종 | 1: Drop(수직 투하), 2: Arc(포물선, 방패 관통용), 3: Freeze(빙결 CC), 4: Pit(함정 CC), 5: Sticky(감속 CC) — 숫자키/클릭으로 선택 |
| 적 3종 | Basic(기본), Shield(방패 — Drop 튕겨냄, Arc만 유효), Runner(고속) — 웨이브에 따라 HP 증가 |
| 오토 모드 | 타겟 중요도(등반 진행도 > 사다리 근접도) 판정, 방패 여부에 따라 무기 자동 전환, 낙하 시간 예측 조준, CC 무기 확률적 사용 |
| 절차적 배경 | 미드포인트 변위 프랙탈 설산 + 랜덤 날씨(맑음/눈/안개) + 건물/나무 배치 |
| 연출 | 파티클, 빙결 시각화(색 변화 + X자 표시), 적 걷기/등반 애니메이션, HP 바 |
| 종료 처리 | 등반 완료 시 게임 오버 오버레이 + 7초 자동 재시작 카운트다운 |

## 발견된 문제점

1. **프레임레이트 의존 게임 루프 (치명적)** — 적 이동, 투사체 물리, 파티클, 이펙트 타이머(`timer: 150` 프레임), 등반 속도가 전부 rAF 틱 단위. 무기 쿨다운·스폰 주기만 `Date.now()` 기반이라 144Hz 모니터에서는 적과 투사체만 2.4배속이 되어 밸런스가 완전히 붕괴됨.
2. **수동 조작이 사실상 없음 (치명적)** — 클릭/스페이스 등 발사 입력 핸들러가 전무. AUTO OFF 상태에서도 발사는 `update()` 안의 자동 정렬 조건(플레이어와 타겟 X 근접 시)으로만 일어나고, 무기 3~5(FREEZE/PIT/STICKY)는 수동으로 발사할 방법 자체가 없음(오토 모드의 프레임당 0.3% 확률 발동뿐). "수동 모드"의 실체가 마우스로 차량 위치만 옮기는 것.
3. **CC 무기가 조준을 무시함** — `fireWeapon(targetX)`로 목표 좌표를 전달해도 cc 계열 분기는 `vx`를 화면 중앙 기준 고정값(±1.5)으로 설정해 목표와 무관한 위치에 떨어짐. 오토 모드의 CC 사용이 낭비가 됨.
4. **시작 화면 데드 코드** — `init()`이 즉시 `startGame()`을 호출하고 `state.screen`도 `'playing'`으로 초기화되어 `#start-screen`이 영원히 표시되지 않음. 첫 방문자가 조작법을 알 기회가 없음.
5. **리사이즈 시 월드 전체 재생성** — resize 이벤트마다 `generateRandomWorld()`가 호출되어 플레이 도중 창 크기만 바꿔도 날씨·산·건물이 통째로 바뀜. 모바일에서 주소창 접힘/회전 시마다 발생.
6. **모바일 미지원** — `mousemove`만 처리하고 터치 이벤트 없음. HUD(좌측 무기 슬롯 65px, 상단 헤더)가 작은 화면 기준 배치 점검이 안 되어 있음.
7. **잔버그**
   - 방패에 튕긴 Drop 투사체가 같은 프레임에 다른 적과 계속 충돌 판정되고, 방패와 겹친 동안 매 프레임 튕김 사운드/파티클 반복 발생 가능
   - 등반 중인 적에게 Freeze/Sticky 이펙트가 X축 거리만으로 적용됨 (지면 이펙트인데 사다리 위 적에게도 작동 — Pit만 `walking` 체크)
   - 게임 오버 화면에서 무기 슬롯 클릭/숫자키가 여전히 동작 (`selectWeapon`에 화면 상태 가드 없음)
   - `updateCooldownUI()`가 매 프레임 5개 슬롯의 DOM style을 무조건 갱신 (변화 없어도 reflow 유발)
8. **게임성 부족** — 최고 점수 저장 없음(localStorage 미사용), 웨이브 시작 알림 없음, 적 처치 점수가 종류 무관 25점 고정, 웨이브가 올라도 적 구성 비율이 항상 균등 랜덤(1웨이브부터 Shield 등장, 후반 긴장감 없음), 플레이어 이동 범위가 좌측 1/3 미달(닿지 않는 구역의 적은 Arc 착탄 대기 외 대응 불가라는 점이 안내되지 않음), 일시정지 없음.
9. **메타데이터 부재** — `<meta name="description">`, Open Graph 태그, 파비콘 없음. `lang="en"`인데 실제 노출 텍스트는 영어라 lang은 유지 가능하나 SEO/공유 품질이 다른 페이지 대비 떨어짐.

## 개선 지시 사항 (→ Sonnet 5 작업)

> 아래 항목은 Sonnet 5 에이전트가 `index.html`을 직접 수정하여 구현한다. 단일 파일·의존성 없음 구조와 기존 비주얼 아이덴티티(설산 다크 테마, BUNTGAMES 브랜딩, 골드 포인트 컬러)는 유지한다.

### 1. 고정 타임스텝 게임 루프 (최우선)
- `requestAnimationFrame` 콜백에서 경과 시간을 누적해 60 updates/sec 고정 타임스텝으로 `update()` 실행 (탭 복귀 시 폭주 방지를 위해 프레임당 최대 업데이트 수 제한)
- 기존 프레임 단위 값(적 speed, 투사체 vy/gravity, 이펙트 `timer: 150`, 파티클 life 등)은 60fps 기준이므로 값 변경 없이 그대로 동작해야 함
- 쿨다운·스폰 주기 등 `Date.now()` 기반 로직은 유지해도 되나, 일시정지(아래 7번) 도입 시 정지 시간이 쿨다운을 갉아먹지 않도록 게임 시간 누적 방식으로 통일 권장

### 2. 수동 조작 완성 (최우선)
- 캔버스 클릭(및 스페이스/엔터 키)으로 현재 선택 무기를 발사: Drop/Arc는 클릭 X좌표를 목표로, CC 무기(3~5)도 클릭 지점으로 투척
- AUTO OFF일 때는 `update()`의 자동 발사 조건을 완전히 비활성화 — 이동·무기 선택·발사가 전부 플레이어 입력이어야 함
- 무기 슬롯에 선택 상태와 별개로 "발사 가능(쿨다운 완료)" 시각 피드백 추가

### 3. CC 무기 조준 수정
- cc 계열 투척도 Arc처럼 `getTravelTime` 기반으로 목표 X좌표에 착탄하도록 `vx` 계산 (수동 클릭 지점 / 오토 모드 타겟 좌표 모두 반영)
- 오토 모드의 CC 사용을 확률 발동 대신 상황 판정으로 개선: 적이 3마리 이상 밀집했거나 Runner가 사다리에 근접했을 때 우선 사용

### 4. 시작 화면 복원 + 모바일 터치 지원
- 첫 로드 시 `#start-screen`을 표시하고 START MISSION 클릭으로 시작 (`state.screen: 'ready'` 초기 상태 추가). 시작 화면에 조작법 요약(키보드/터치) 표시
- 터치 디바이스: 터치 드래그로 케이블카 이동(AUTO 자동 해제), 탭으로 발사, 무기 슬롯 터치 타깃 44px 이상 확보
- resize 시 월드 재생성 금지 — 배경은 기존 시드 데이터로 다시 그리기만 하고(산 좌표는 폭 비율로 재배치 또는 단순 재스케일), 날씨는 유지. 월드 재생성은 `startGame()`에서만
- 작은 화면(375px)에서 헤더/스탯바/무기 슬롯이 겹치지 않도록 배치 조정

### 5. 잔버그 수정
- 방패 튕김 시 해당 투사체에 짧은 무적(재충돌 금지) 시간 부여, 같은 프레임 다중 판정 방지
- Freeze/Sticky 이펙트는 `walking` 상태의 적에게만 적용 (Pit과 동일 기준). 등반 중인 적을 막으려면 Freeze만 예외적으로 허용할지 결정하고 일관되게 처리
- `selectWeapon`·`fireWeapon`에 `state.screen !== 'playing'` 가드 추가
- `updateCooldownUI`는 값이 변한 슬롯만 DOM 갱신

### 6. 게임성 강화
- **최고 점수**: localStorage에 최고 웨이브/점수 저장, 헤더와 게임 오버 화면에 표시
- **웨이브 연출**: 웨이브 시작 시 "WAVE N" 배너 (1~1.5초), 웨이브 클리어 보너스 점수
- **점수 차등화**: Basic 25 / Runner 40 / Shield 50, 등반 중인 적 처치 시 1.5배
- **적 구성 커브**: 1~2웨이브는 Basic만, 3웨이브부터 Runner, 5웨이브부터 Shield 등장, 이후 웨이브에 따라 비율 점진 변화. 8웨이브 이후 신규 변형 1종 추가(예: 저속·고HP 탱커)
- **콤보 시스템(선택)**: 3초 내 연속 처치 시 점수 배율 (HUD 표시)

### 7. 편의 기능·마무리
- P키 + HUD 버튼으로 일시정지/재개 (오버레이 표시, 모든 이동·타이머·사운드 정지)
- `<meta name="description">`, Open Graph 태그(title/description/type), 인라인 SVG 파비콘 추가

## 검증 체크리스트 (1차)
- [x] 60Hz/144Hz에서 적·투사체 속도 동일
- [x] AUTO OFF에서 클릭으로 5종 무기 전부 발사 가능, 자동 발사는 완전 정지 → **2차에서 방향 수정됨 (아래 참조)**
- [x] CC 무기가 클릭/타겟 지점에 착탄
- [x] 첫 로드 시 시작 화면 표시 → **2차에서 방향 수정됨 (즉시 시작으로 복원)**
- [x] 플레이 중 창 리사이즈 시 날씨/지형 유지
- [x] 게임 오버 화면에서 무기 조작 불가, 최고 점수 저장·표시
- [x] 일시정지 중 모든 이동·쿨다운·사운드 정지, 재개 시 쿨다운 손실 없음

# 2차 개선: 게임의 재미 재정의

## 사용자 피드백 (1차 결과에 대한)
1. 페이지 진입 시 바로 시작해야 하는데 시작 화면에서 멈춰 있음
2. 자동 사냥을 구경하는 재미가 사라짐 — 적이 사다리에 도달해서야 공격이 시작됨
3. 기존의 Arc 고정 오프셋 제약은 의도된 설계였음: 자동 발사를 전제로 **위치 이동만 하면 되는** 조작 단순화

## 재미 분석: 이 게임은 무엇으로 재미있어야 하는가

이 게임의 정체성은 **관전형(방치형) 디펜스**다. BizCard의 attract 모드처럼 페이지를 열어두면 스스로 돌아가는 데모이자, 원하면 가볍게 개입할 수 있는 게임. 재미의 축은 세 가지:

1. **구경의 재미 (핵심)** — 유능한 AI가 필드 전역에서 적을 요격하는 모습을 지켜보는 것. 케이블카가 부지런히 오가며 걷는 적을 낙하 예측 사격으로 잡아내고, 방패병에겐 Arc로 전환하고, 밀집 지점에 CC를 던지는 "일 잘하는 기계"의 그림. 사다리 앞에서만 몰아치는 현재 패턴은 이 재미를 죽임.
2. **개입의 재미** — 낮은 조작량, 높은 영향. 마우스로 위치를 옮겨 우선순위를 바꾸고, 결정적 순간에 CC를 원하는 지점에 던지는 것. 조준·연사 같은 고밀도 조작은 이 게임의 재미가 아님 (1차의 "완전 수동화"는 과잉이었음).
3. **긴장의 재미** — 웨이브 상승에 따른 압박, 적이 사다리에 붙었을 때의 위기 방어. 단, 위기는 예외 상황이어야지 기본 패턴이면 안 됨.

### "사다리에 와서야 공격"의 기술적 원인
오토 로직에서 **이동 목표의 리드**(`target.x + target.speed * t`, t = 낙하 시간 ≈ 70프레임)와 **발사 판정의 리드**(`target.x + target.speed * 15`, 허용 오차 25px)가 서로 다른 예측 지점을 사용한다. Basic(speed 0.8) 기준 두 지점의 차이가 약 44px로 허용 오차를 항상 초과 → 걷는 동안엔 발사 조건이 성립하지 않고, 적이 사다리에 붙어 X가 고정된 후에야(`pX = fireX = ladderX`) 조건이 일치해 발사가 시작된다. (이 불일치는 원본 코드부터 있었고, 1~2웨이브가 Basic 전용이 되면서 더 두드러짐.)

## 개선 지시 사항 2차 (→ Sonnet 5 작업)

### 1. 즉시 시작 복원 (최우선)
- 시작 화면 게이트 제거 — 로드 즉시 플레이 시작 (attract 모드 정체성). 게임오버 7초 자동 재시작 유지로 방치 시 무한 데모처럼 순환
- 조작법 안내는 시작 화면 대신 게임 시작 직후 3~4초 표시 후 페이드아웃되는 힌트 토스트로 대체 (키보드/터치 환경별 문구)

### 2. 오토 사격을 필드 전역 요격으로 (최우선)
- 이동 목표와 발사 판정이 **동일한 요격 지점**(낙하 시간 기반 intercept)을 쓰도록 통일
- 발사 판정을 "현재 발사 시 예측 착탄 지점이 적 예상 위치의 히트 반경 이내"로 변경, 쿨다운이 돌 때마다 기회 사격
- 최중요 타겟만이 아니라 이동 경로상 정렬된 아무 적에게나 기회 사격(opportunistic fire) 허용 — 화면 전역에서 교전이 벌어지는 그림
- 검증 기준: 오토 모드에서 대부분의 적이 필드 중반 이전에 처치되고, 사다리 등반은 예외적 위기 상황이어야 함

### 3. 수동 모드 재정의: "위치만 조작" 복원
AUTO OFF의 의미를 "조준·발사까지 수동"에서 **"위치 조작만 수동, 발사는 자동"**으로 되돌린다:
- 마우스 이동/터치 드래그 = 케이블카 위치 지정 (기존 유지)
- Drop/Arc는 정렬 시 자동 발사 — 2번의 통일된 intercept 판정 재사용, Arc 고정 오프셋(±180) 거리 유지 (오프셋 자체가 포지셔닝 재미)
- 클릭/탭 = **선택된 CC 무기(3~5)를 해당 지점에 투척** (Drop/Arc 선택 중 클릭은 무시 또는 즉시 1회 강제 발사 — 구현 판단에 맡김)
- 숫자키/슬롯 클릭 무기 선택, 스페이스/엔터 강제 발사는 유지

### 4. 관전 재미 강화
- 오토 모드가 CC를 쓸 때 해당 무기 슬롯 하이라이트 등 시각 신호 (AI의 판단이 보이도록)
- 처치 점수/콤보 팝업이 처치 위치에서 떠오르게 (화면 곳곳에서 교전 피드백)
- 적이 사다리 등반을 시작하면 경고 연출 (사다리 주변 붉은 펄스 등) — 과하지 않게

### 5. 회귀 금지
고정 타임스텝, 일시정지, 최고점수 저장, 모바일 레이아웃, 리사이즈 시 지형 유지, 웨이브 커브/점수 차등 등 1차 개선 사항은 모두 유지한다.
