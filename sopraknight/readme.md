# SopraKnight

SopraKnight is a single-file prototype for a lane-based concert hall defense game.

The core idea is not to defeat enemies. Emotionless undead enter the hall, and musicians restore their emotions through music until they become audience members.

## Current Build

- Entry file: `sopraknight/index.html`
- No build step
- No backend
- Runs as a plain browser HTML file
- Uses HTML5 Canvas for the game field
- Uses local PNG/JPG assets from `sopraknight/images`

## Game Concept

Working title:

`SopraKnight: Defend the Concert Hall`

Tagline:

`The world does not need another battle. It needs one final performance.`

The prototype currently follows a Plants vs Zombies style structure:

- 6 horizontal lanes
- Undead enter from the right and move left
- Musicians are placed on stage slots
- Musicians automatically perform
- Music restores hope and reduces despair
- Restored undead become audience members
- Audience generates inspiration
- If too many undead reach the stage, morale falls

## Player Flow

Current default mode is automatic demo mode.

1. The page opens.
2. Auto mode randomly chooses an ensemble.
3. One musician is placed every 1 second.
4. After placement is complete, the concert starts automatically.
5. The round plays out.
6. When the round ends, a 7 second countdown starts.
7. The game resets and repeats with a new random ensemble.

Manual interaction still exists:

- Click a musician portrait on the left panel.
- Click a glowing stage slot.
- Press `Begin Concert`.
- Press `Reset` to restart.
- Press `Grand Wave` during play for a global skill.

## Main Systems

### Lane System

Defined in `index.html`:

- `lanes`
- `stageTiles`

There are currently 6 lanes. Each lane has 4 stage slots.

### Musician Types

Defined in `musicianTypes`.

Current classes:

- Soprano: adjacent-lane area resonance
- Pianist: same-lane support attack
- Cellist: adjacent-lane slow field
- Flutist: same-lane push/control
- Trumpeter: same-lane piercing attack
- Harpist: adjacent-lane morale/healing aura

Important fields:

- `id`
- `name`
- `role`
- `cost`
- `range`
- `rate`
- `hope`
- `despair`
- `color`
- `group`

### Undead Types

Defined in `undeadTypes`.

Current types:

- Shambler
- Runner
- Elite
- Brute

Important fields:

- `speed`
- `hope`
- `radius`
- `spriteSize`
- `morale`
- `color`

### Range Logic

Range logic is centralized in:

- `rangeProfile(type, origin)`
- `enemiesInRange(type, origin)`

This is important because both attack behavior and range preview use the same range profile.

Do not duplicate range logic in drawing code or attack code.

### Visual Attack Feedback

When a musician attacks, the game creates a short beam effect:

- `emitBeam(from, to, color)`
- stored in `state.beams`

This makes it visible which undead was targeted.

### Status Display

Undead do not use HP bars.

This was intentional. Earlier green/red bars looked like health increasing or decreasing, which fought the theme.

Current status style:

- despair is shown as a small fading gray arc near the feet
- hope is shown only as a subtle cyan aura

Future versions should keep the language of restoration, not damage.

## Auto Mode

Auto mode is controlled by state fields:

- `autoMode`
- `autoPhase`
- `autoTimer`
- `autoSlotIndex`
- `restartCountdown`

Random placement is generated by:

- `buildAutoPlacementPlan()`

Current auto behavior:

- guarantees at least one musician per lane
- randomizes lane slot
- randomizes musician class
- adds 1 to 3 extra random musicians
- starts the concert automatically
- restarts after a 7 second countdown

The concert hall background is drawn in code:

- `drawFutureHallBackdrop()`

It includes:

- dark gradient hall
- gold arch structures
- subtle futuristic cyan lines
- spotlights
- reflective floor

The stage and audience areas are also drawn directly on canvas.

## Known Issues / Design Notes

- Current musician assets are portrait-card style, not full body sprites.
- The game is readable, but the tactical depth is still light.
- Auto mode is useful for demo loops but should eventually be toggleable.
- Enemy group composition is fixed in `undeadPlan`.
- There is no level system yet.
- There is no persistent settings storage.
- There is no sound.
- The current balance is only rough.
- The UI is optimized mainly for desktop.

## Important Design Principle

This game should feel like healing, performance, and emotional restoration.

Avoid making it feel like a shooter or damage-focused combat game.

Prefer terms and visuals like:

- hope
- despair fading
- resonance
- restoration
- audience
- morale
- applause
- performance

Avoid:

- HP bars
- blood
- damage numbers
- kill language
- gore
