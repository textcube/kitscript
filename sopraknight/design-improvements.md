# SopraKnight 디자인 개선안

현재 빌드(`sopraknight/index.html`, 단일 파일 프로토타입)를 기준으로 작성한 시각/UX 디자인 개선 제안서입니다.

"치유와 공연, 감정 회복"이라는 핵심 설계 원칙(readme.md 참조)을 유지하면서, 가독성·게임필·연출을 끌어올리는 방향으로 구성했습니다.

---

## 1. 현재 상태 진단

### 잘 되어 있는 부분

- 골드/네이비/시안 팔레트가 "미래형 클래식 콘서트홀" 컨셉과 일치
- 사거리 로직이 `rangeProfile()`로 중앙화되어 공격과 미리보기가 항상 일치
- HP바 대신 절망(회색 호) / 희망(시안 오라)을 쓰는 상태 표현이 테마에 충실
- 빔 이펙트(`emitBeam`)로 타겟팅이 시각적으로 명확
- Web Audio 기반 사운드 토글이 이미 구현됨 (readme의 "There is no sound"는 갱신 필요)

### 개선이 필요한 부분

- 언데드의 회복 진행도(hope)가 은은한 오라뿐이라 "얼마나 더 연주하면 관객이 되는지" 읽기 어려움
- 회복 → 관객 전환 순간이 게임의 핵심 판타지인데 연출이 약함 (파티클 18개 + 토스트뿐)
- 오토 모드를 끌 방법이 UI에 없음 (readme에도 next step으로 명시)
- 배치된 음악가의 쿨다운/연주 상태가 보이지 않음
- 색상 값이 CSS 변수와 JS 하드코딩 hex로 이중 관리됨
- 초기 HTML의 `0 / 36` 표기와 실제 `targetAudience: 30`(좌석 2×15=30)이 불일치
- `Concert 0 / 1` 스탯은 원-콘서트 챌린지 구조에서 정보 가치가 낮음

## 2. 개선안 (우선순위순)

### A. 감정 회복 가독성 — 최우선

핵심 루프가 "언데드를 회복시켜 관객으로 만든다"이므로, 진행도가 한눈에 보여야 합니다.

- **희망 게이지 강화**: 현재 발밑 절망 호(gray arc)와 대칭으로, 머리 위에 작은 **희망 호**를 추가. 회복이 진행될수록 회색 → 시안 → 골드로 색이 밝아지며 채워짐. "체력이 닳는다"가 아니라 "빛이 차오른다"는 언어 유지.
- **회복 임박 연출**: hope가 80% 이상이면 스프라이트에 골드 림라이트 + 작은 음표(♪) 파티클이 주기적으로 떠오름. 플레이어가 "저 언데드는 곧 관객이 된다"를 즉시 인지.
- **전환 연출 강화**: 관객 전환 순간에 (1) 흰색 플래시 확산, (2) 음표 파티클 상승, (3) 좌석 도착 시 좌석이 골드로 점등 + 살짝 바운스. `restoreEnemy()` 한 곳만 수정하면 됨.

### B. 음악가 상태 표시

- **쿨다운 인디케이터**: 배치된 음악가 발밑 그림자 위에 얇은 원형 게이지로 다음 연주까지 남은 시간 표시 (`m.cooldown / m.type.rate`).
- **연주 애니메이션**: 연주 시점에 스프라이트를 1프레임 스케일 펄스(1.0 → 1.08 → 1.0) + 악기 색 글로우. 별도 스프라이트 없이 코드로 가능.
- **대기 애니메이션**: `Math.sin(time + offset)` 기반의 미세한 상하 흔들림. 정적인 무대에 생동감 부여, 비용 거의 없음.

### C. 오토/수동 모드 UI

- **quick-actions 줄에 Auto 토글 버튼 추가** (사운드 버튼과 동일한 `icon-btn` 패턴). 켜짐 상태는 `sound-on`처럼 시안 하이라이트.
- 오토 모드 중임을 캔버스에 표시: 상단 메시지 옆에 "AUTO" 배지. 수동 개입(카드 클릭 등) 시 오토 모드 자동 해제 여부는 게임 디자인 결정 필요.
- **배속 컨트롤(1x/2x)**: 데모 루프와 테스트 모두에 유용. `dt`에 배수만 곱하면 됨.

### D. HUD / 좌측 패널

- **Morale을 숫자 → 게이지 바**로: 위기감이 수치보다 직관적으로 전달됨. 낮을수록 골드 → 앰버 → `--danger` 색 전이.
- **Audience를 목표 진행 바**로: `0 / 30`과 함께 채워지는 바. 승리 조건이 항상 시야에 있게.
- **Concert 스탯 대체**: `0 / 1` 대신 "남은 언데드 수"(`spawnLeft + enemies.length`)를 표시하면 라운드 종반 긴장감이 생김.
- 초기 HTML의 `0 / 36`을 `0 / 30`으로 수정 (좌석 수와 일치).
- 음악가 카드에 사거리/속도 미니 아이콘 또는 1줄 스탯 추가 (현재는 역할 텍스트뿐이라 비교 선택이 어려움).

### E. 캔버스 연출 (Stage Presence)

- **커튼 연출**: 콘서트 시작 시 골드 커튼이 좌우로 열리고, 종료 시 닫힘. 오토 모드의 7초 카운트다운과 자연스럽게 연결됨.
- **스포트라이트 반응**: `drawFutureHallBackdrop()`의 3개 조명이 현재 연주 중인 음악가 쪽으로 은은하게 기울거나 밝아짐.
- **Grand Wave 연출 강화**: 현재 단일 펄스 → 홀 전체를 쓸고 가는 빛의 파도 + 레인 라이트가 순차 점등. readme next step("stage lighting that reacts to Grand Wave")과 일치.
- **바닥 반사**: 스프라이트를 뒤집어 저알파로 그리는 값싼 반사 효과. "reflective floor" 컨셉 강화.

### F. 위기 표현 (테마 준수)

언데드가 무대에 도달할 때 "피해"가 아니라 "공연이 위협받는다"는 언어로:

- 모랄 하락 시 해당 레인의 조명이 잠시 어두워지고 무대 테두리가 흐려짐
- 모랄 30 이하일 때 홀 전체에 은은한 비네트(어두운 가장자리) — 붉은 플래시나 흔들림 같은 전투 게임 문법은 피함
- 무거운 웨이브 진입 전 해당 레인 끝에 경고 글로우 (readme next step "lane-specific warnings")

### G. 코드/디자인 토큰 정리

- 색상을 단일 소스로: CSS `:root` 변수와 JS 상수(`#f5c86a` 등 하드코딩)를 하나의 팔레트 객체로 통합. 예: JS에 `PALETTE` 상수를 두고 CSS 변수는 시작 시 주입하거나, 최소한 이름 있는 상수로 정리.
- 시맨틱 토큰 정의: `--hope`(시안), `--gold`, `--despair`(회색), `--inspiration`, `--morale` — 이후 밸런싱/리스킨 시 한 곳만 수정.

### H. 접근성 / 반응형

- `.stat span`(0.58rem) 등 극소 텍스트의 대비·크기 재점검
- `prefers-reduced-motion` 존중: 파티클/펄스 밀도 축소 모드
- 모바일(≤880px)에서 음악가 카드 터치 타겟 확대, 캔버스 탭 배치 정밀도 개선 (현재 히트 반경 56px은 축소된 캔버스에서 체감상 작아짐)

## 4. 제안 우선순위 요약

| 순위 | 항목 | 효과 | 구현 난이도 |
|---|---|---|---|
| 1 | A. 감정 회복 가독성 | 핵심 루프 이해도 | 낮음 |
| 2 | B. 음악가 상태 표시 | 전술 판단 근거 | 낮음 |
| 3 | C. 오토/수동 토글 + 배속 | 데모·테스트 편의 | 낮음 |
| 4 | D. HUD 게이지화 | 위기감·목표 인지 | 낮음 |
| 5 | E. 커튼/조명 연출 | 공연 판타지 몰입 | 중간 |
| 6 | F. 위기 표현 | 긴장감 (테마 준수) | 중간 |
| 7 | G. 디자인 토큰 정리 | 유지보수성 | 낮음 |
| 8 | H. 접근성/반응형 | 도달 범위 | 중간 |

1~4번은 모두 기존 단일 파일 구조 안에서 작은 수정으로 가능하며, 스프라이트 추가 작업이 필요 없습니다.
