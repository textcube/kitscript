# Eternal Invasion: Puzzle Edition

7x10 보드에서 몬스터 블록이 주기적으로 내려오고, 하단의 마법사가 자동으로 가장 위험한 열을 추적해 발사하는 관전형 매치 퍼즐 방어 게임입니다. 플레이어는 인접한 블록을 스왑해 3개 이상 연결된 군집을 만들거나, `Salvo` 스킬로 전 열을 일제 사격해 생존 시간을 늘립니다.

게임은 `index.html` 하나로 실행됩니다.

## 현재 구조

- 단일 HTML 파일: 마크업, CSS, JavaScript가 모두 `index.html`에 포함되어 있습니다.
- 렌더링: 7x10 보드를 DOM `div` 70개로 그립니다.
- 게임 루프: `requestAnimationFrame` 기반 루프에서 AI 이동, 자동 사격, 몬스터 하강, 발사체 이동을 처리합니다.
- 사운드: Web Audio API로 선택, 발사, 피격, 스킬, 경보음을 합성합니다.
- 진행 방식: 하강 성공 시 `wave`가 증가하고 하강 간격은 점점 짧아집니다.
- 기록: 최고 점수는 `localStorage`에 저장됩니다.

## 주요 게임 규칙

| 항목 | 설명 |
| --- | --- |
| 보드 | 7열 x 10행 |
| 초기 배치 | 상단 4행에 몬스터 생성 |
| 자동 공격 | 마법사가 가장 아래까지 내려온 몬스터 열을 추적하고 주기적으로 발사 |
| 매치 처리 | 발사체가 닿은 블록과 같은 타입의 연결 군집이 3개 이상이면 군집 전체 제거 |
| 단독 제거 | 군집이 3개 미만이면 피격 블록만 제거 |
| 콤보 에너지 | 제거 누적 100마다 `Salvo` +1, 최대 10 |
| 방어막 | 최하단이 막힌 상태에서 하강 타이밍이 오면 25 감소 |
| 게임오버 | 방어막이 0이 되면 결과 화면 표시 후 7초 뒤 자동 재시작 |

## 이번 개선 사항

- 게임오버 후 자동 재시작과 Retry 클릭이 겹치며 `requestAnimationFrame` 루프가 중복 실행되던 문제를 수정했습니다.
- 재시작 시 `projectiles`, `selectedCell`, 마법사 위치, 타이머 기준값, 이펙트 상태를 모두 초기화합니다.
- 재시작 카운트다운 `setInterval`을 단일 핸들로 관리해 이전 타이머가 새 게임에 개입하지 않게 했습니다.
- 발사체 이동을 프레임 단위가 아니라 delta time 기준으로 바꿔 60Hz와 144Hz 화면에서 속도 차이가 커지지 않게 했습니다.
- 호출만 있던 `select` 사운드 분기를 추가했습니다.
- 같은 블록을 다시 클릭하면 선택이 해제되며, 중력 정렬 후 오래된 선택 좌표가 남지 않습니다.
- `@apply`에 의존하던 `.btn-social` 스타일을 일반 CSS로 바꿔 Tailwind Play CDN 처리 여부와 관계없이 버튼이 정상 표시됩니다.
- 최고 점수와 웨이브 표시를 HUD에 추가했습니다.
- `meta description`, Open Graph 메타 태그, 버튼 `aria-label`, 작은 화면 높이 보정을 추가했습니다.

## 남은 개선 후보

- Tailwind Play CDN을 완전히 제거하고 정적 CSS만으로 정리하면 오프라인 실행 안정성이 더 좋아집니다.
- 보드 DOM을 매번 재생성하지 않고 셀을 재사용하면 장시간 실행 성능이 개선됩니다.
- 발사체 DOM도 풀링하거나 `transform`만 갱신하도록 바꾸면 프레임별 DOM churn을 줄일 수 있습니다.
- 자동 마법사 AI를 “가장 낮은 블록”뿐 아니라 “3개 이상 군집을 만들 수 있는 열”까지 고려하도록 개선할 수 있습니다.
- 게임오버 결과 화면에 생존 시간, 도달 웨이브, 최대 매치 크기를 추가하면 기록성이 좋아집니다.
- 접근성 측면에서 키보드 조작과 포커스 스타일을 추가할 수 있습니다.

## 검증 체크리스트

- [ ] 게임오버 카운트다운 중 Retry를 눌러도 자동 재시작이 한 번만 발생한다.
- [ ] 게임오버를 여러 번 반복해도 발사 속도와 렌더링 속도가 빨라지지 않는다.
- [ ] 새 게임 시작 시 이전 발사체와 선택 링이 남지 않는다.
- [ ] 60Hz와 144Hz 디스플레이에서 발사체 도달 시간이 크게 달라지지 않는다.
- [ ] `Salvo` 버튼이 스킬 수 0일 때 비활성처럼 보이고, 1 이상일 때 강조된다.
- [ ] 최고 점수가 브라우저 새로고침 후에도 유지된다.
- [ ] 375x667 크기에서 주요 UI가 겹치지 않는다.

## 실행 방법

브라우저에서 `puzzle/index.html` 파일을 열면 바로 실행됩니다. 로컬 서버를 쓰고 싶다면 저장소 루트에서 정적 서버를 띄운 뒤 `/puzzle/` 경로로 접속하면 됩니다.
