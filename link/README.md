# Local Message Desk (LMD) 설계 초안 v0.1

## 목표

브라우저에서 실행되는 **1페이지 로컬 메시지 작업 도구**를 만든다.

-   서버 없음
-   회원가입 없음
-   모든 데이터는 로컬 저장(IndexedDB)
-   연락처, 템플릿, 작업 상태를 관리
-   실제 발송은 사용자가 문자/메신저 앱에서 수행
-   발송 이력과 진행 상황을 관리하는 것이 핵심

------------------------------------------------------------------------

# 핵심 개념

    Campaign
        └── Group(Tag)
                └── Contact
                        └── Action
                                └── Result

-   Campaign : 이번 작업(공연 안내, 행사 초청 등)
-   Group : VIP, 운영진, 후원자 등
-   Contact : 연락처
-   Action : 실제 수행한 작업
-   Result : 작업 결과

## Group(Tag)

-   name
-   color

한 사람이 여러 그룹에 속할 수 있다.

------------------------------------------------------------------------

## Action

한 번의 작업을 의미한다.

예)

-   문자 발송
-   카카오톡
-   단체 메시지
-   이메일

필드

-   id
-   campaignId
-   channel
-   contactIds\[\]
-   createdAt

------------------------------------------------------------------------

## Result

각 연락처별 상태

-   미처리
-   처리중
-   발송완료
-   발송후보류
-   미발송보류
-   오류
-   응답
-   완료
-   제외

추가 정보

-   updatedAt
-   reason
-   memo
