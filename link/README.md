# Local Message Desk (LMD) 설계 초안 v0.2

## 목표

브라우저에서 실행되는 **1페이지 로컬 메시지 작업 도구**를 만든다.

-   서버 없음
-   회원가입 없음
-   모든 데이터는 로컬 저장(IndexedDB)
-   연락처, 템플릿, 작업 상태를 관리
-   실제 발송은 사용자가 문자/메신저 앱에서 수행
-   발송 이력과 진행 상황을 관리하는 것이 핵심

------------------------------------------------------------------------

# 핵심 개념

    Campaign
        └── Group(Tag)
                └── Contact
                        └── Action
                                └── Result

-   Campaign : 이번 작업(공연 안내, 행사 초청 등)
-   Group : VIP, 운영진, 후원자 등
-   Contact : 연락처
-   Action : 실제 수행한 작업
-   Result : 작업 결과

# v0.2 추가 사항

-   테마 토글 : 시스템/라이트/다크 3단계, `localStorage('lmd-theme')` 저장
-   연락처 삭제 : 상세 패널에서 삭제(관련 상태 기록과 이력 정리 포함)
-   캠페인 편집·삭제 : 제목 옆 버튼으로 수정, 삭제 시 상태 기록·이력 함께 제거
-   가져오기 중복 병합 : 전화번호(숫자 정규화)·이메일(소문자) 일치 시 기존 연락처에 병합
-   캠페인 타깃 그룹 : `targetTags`로 캠페인 대상 연락처 제한(목록·통계 반영)
-   실행 취소 : 제외/미루기/일괄 상태 변경 직후 토스트에서 되돌리기
-   전체 데이터 초기화 : 2단계 확인 후 로컬 데이터 전부 삭제
-   키보드 단축키 : ↑/↓ 이전·다음 연락처, Enter 문자 보내기(데스크톱)
-   버그 수정 : vCard 중복 키 크래시, iOS `sms:` 구분자, 제외 시 메모 보존, 연락처 정보 없는 액션 버튼 비활성화, 필터 변경 시 선택 초기화
-   모바일 캠페인 레일·빠른 필터 칩(760px 이하) : 사이드바 nav가 숨겨지는 모바일 화면에서 캠페인 목록·전체 연락처·새 캠페인(＋)을 가로 스크롤 칩으로 노출하고, 상태별(미처리/발송완료/보류/오류/완료) 빠른 필터 칩을 별도 줄로 제공(재탭 시 필터 해제)
-   모바일 제목 스와이프 전환 : 제목 블록을 좌우로 스와이프해 [전체 연락처, ...캠페인] 순서로 캠페인을 순환(양끝은 정지 + 피드백 메시지)
-   모바일 첫 화면 밀도 개선(760px 이하) : 상단 액션을 "가져오기·연락처·더보기" 한 줄로 줄이고 나머지 6개 작업(테마·데모·붙여넣기·복사·내보내기·초기화)은 "더보기" 바텀시트로 이동, 통계 4카드를 슬림 한 줄 바로 축소, 브랜드 바 여백 축소로 첫 연락처 행이 스크롤 없이 보이도록 함

# v0.3 추가 사항 — 국제화(i18n)

-   영어/한국어 2개 언어, 기본값은 **영어**. 브랜드 영역의 배지 버튼("EN"/"한")을 클릭할 때마다 즉시 전환되며 `localStorage('lmd-lang')`에 저장되어 다음 방문에도 유지된다. 리로드 없이 화면 전체 텍스트가 갱신된다.
-   `STATUSES` 배열(연락처 상태의 IndexedDB 저장값)은 언어와 무관하게 계속 한글 문자열을 내부 식별자로 사용한다. 화면에는 `STATUS_LABEL[lang][내부값]` 매핑을 거친 라벨만 표시하므로, 필터·배지·드롭다운의 표시 텍스트가 바뀌어도 저장된 데이터와 상태 비교 로직은 전혀 영향받지 않는다. 채널(문자/전화/이메일/공유)도 같은 방식으로 `CHANNEL_LABEL`을 둔다.
-   연락처 이름·전화번호·메모·태그, 캠페인 제목·설명·커스텀 템플릿 등 **사용자 데이터는 번역하지 않는다.** 오직 버튼·라벨·안내 문구 같은 UI 크롬만 번역 대상이다.
-   데모 데이터(`loadDemo()`)는 언어별로 별도 데이터셋(`DEMO_DATA.en`/`DEMO_DATA.ko`)을 사용한다. 연락처·캠페인·상태 기록의 `id`는 언어와 무관하게 고정되어 있어, 다른 언어로 다시 불러와도 같은 레코드가 정상적으로 덮어써진다.
-   자세한 구현 방식은 `DESIGN.md`의 "국제화(i18n)" 절을 참고.

이후 실제 사용하면서 UI와 상태 체계를 계속 다듬는다.
