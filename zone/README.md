# Eternal Invasion: Zone Edition

Qix/Volfied 계열 땅따먹기 캔버스 게임. 100×75 그리드에서 트레일을 그려 폐곡선을 완성하면 빈 영역 중 가장 큰 곳을 제외한 나머지가 점령되고, 점령률 80% 달성이 목표다. 필드 안에는 탄성 충돌 물리로 튕겨 다니는 적 3기가 있으며 트레일에 닿으면 목숨을 잃는다. 페이지 진입 즉시 "AUTO HUNTING" AI가 자동으로 영토를 넓히는 관전형(attract) 게임이며, 방향키/D-패드 입력으로 언제든 제어를 넘겨받고 5초 무입력 시 AI가 복귀한다. 게임오버/승리 후 7초 자동 재시작. 단일 파일(`index.html`) 구성.

## 현재 구조 분석

### 기술 스택
- **단일 HTML 파일** (약 640줄) — 외부 의존성: **Tailwind Play CDN** + Google Fonts(Press Start 2P)
- **렌더링**: Canvas 2D 고정 600×450(6px 셀) + 점령지 오프스크린 캔버스 캐시 + 절차적 배경 패턴(무작위 원), CRT 스캔라인 CSS 오버레이
- **오디오**: Web Audio API 자체 신스(이동/점령/적 처치/피격/승리 5종), 기본 OFF
- **상태 관리**: `Game` 클래스 + 2차원 그리드(0=빈 땅, 1=점령, 2=트레일), 적/파티클 배열
- **게임 루프**: `requestAnimationFrame` 직결, 이동·물리 전부 프레임 단위 (플레이어 4프레임/칸, 터보 2프레임/칸)

### 주요 기능
| 기능 | 설명 |
|------|------|

| 종료 처리 | 80% 승리 또는 목숨 소진 → 오버레이 + 7초 카운트다운 자동 재시작 + 수동 재시작 버튼 (rAF 루프 중복 방지 가드 있음) |

## 발견된 문제점

1. **프레임레이트 의존 게임 루프 (치명적)** — 플레이어 이동(4프레임/칸), 적 속도(0.35~0.48칸/프레임), 파티클 수명이 전부 rAF 틱 단위. 144Hz 모니터에서 게임 전체가 2.4배속으로 돌아감. 유휴/스폰 점멸 등 일부만 `Date.now()` 기반으로 시간 축도 이원화되어 있음.
2. **모바일 레이아웃 파탄 (치명적)** — 캔버스가 고정 600×450px인데 `max-width` 축소 규칙이 없고 `body`가 `overflow: hidden`이라, 375px 화면에서는 **전장 오른쪽 ~40%가 잘린 채 스크롤도 불가**. D-패드는 표시되지만 화면 밖으로 밀려나면 접근 불가.
3. **모바일 D-패드가 오토 모드를 해제하지 못함 (치명적 버그)** — `disableAutoMode()`는 키보드 경로(`setDirectionFromKey`)에서만 호출되고, D-패드의 `setDirection(dx, dy, true)`는 `lastManualInputTime`만 갱신함. `isAutoMode`가 계속 true라 AI가 매 프레임 방향을 덮어써 **터치 조작이 사실상 불능** — AI 결정 사이사이에만 입력이 반짝 먹히는 잭 상태.
4. **스폰 점멸 ≠ 무적 (오해 유발)** — 스폰 후 2초 점멸 연출은 있지만 실제 무적 시간은 없음. 적이 리스폰 지점 근처에 있으면 즉시 연속 피격 가능하고, `moveEnemies`의 forEach에서 적마다 `playerHit()`을 호출해 한 프레임에 목숨 2 이상 잃을 수도 있음. 또 `lastSafeX/Y`가 점령 완료 시점에만 갱신되어, 경계를 따라 멀리 이동한 뒤 피격되면 엉뚱하게 먼 곳으로 리스폰됨.
5. **안전지대 없는 근접 피격** — 거리 1.5칸 근접 판정이 드로잉 여부와 무관하게 상시 적용되어, 점령지/경계 위에 가만히 서 있어도 벽에 튕긴 적이 스치면 피격됨. Qix 계열 문법(점령지 위는 안전, 트레일만 위험)과 어긋나고, 관전(방치) 중 억울한 죽음이 발생함.
6. **드로잉 중 역방향 입력 허용** — 역방향 차단 가드가 `!isDrawing`일 때만 동작해, 트레일을 그리는 중 반대 방향을 누르면 자기 트레일을 밟고 즉시 피격됨.
7. **적 반사 물리 부정확** — 벽/점령지 충돌 시 침투 축 구분 없이 같은 셀 조건으로 vx·vy가 **동시에** 반전되어(수직 반사가 아닌 정반사) 적이 점령지 벽을 따라 미끄러지지 못하고 항상 대각으로 되튕김. 반전 후 `±v×2` 점프 보정이 다른 벽으로 파고들거나 코너 끼임을 유발할 수 있음.
8. **진행·점수 설계 빈약** — 점수원이 적 처치(+20,000)뿐이고 **점령 면적은 0점**. 레벨/웨이브 개념이 없어 80% 달성 후 같은 난이도로 재시작만 반복되고, 폐곡선으로 적을 3기 모두 잡으면 위협 0인 채로 남은 땅을 무의미하게 채움. localStorage 최고 기록도 없음.
9. **Tailwind Play CDN 의존** — 프로덕션 비권장(콘솔 경고, 런타임 JIT 로드), 오프라인 동작 불가. 형제 게임들의 "의존성 없는 단일 파일" 원칙과 어긋남.
10. **잔버그·폴리시**
    - 두 번째 게임오버 오버레이에 직전 판의 "Restarting in 0s..." 문구가 1초간 잔류(카운트다운 초기화 누락)
    - `AudioContext`를 사용자 제스처 전에 생성 — 브라우저 자동재생 정책 경고
    - D-패드/터보의 `onmousedown`+`ontouchstart` 병기로 터치 시 이벤트 중복 발화 (`preventDefault` 없음)
    - 수동 조작 중 5초 멈춰 생각하면 AI가 제어를 탈취 — 드로잉 도중이면 AI가 갑자기 PANIC 귀환을 시도해 의도가 깨짐
    - `<meta name="description">`, Open Graph 태그, 파비콘 없음

## 개선 지시 사항 (→ Sonnet 5 작업)

> 아래 항목은 Sonnet 5 에이전트가 `index.html`을 직접 수정하여 구현한다. 단일 파일 구조와 기존 비주얼 아이덴티티(레트로 픽셀/CRT, Press Start 2P, 네온 시안·마젠타)는 유지한다.
>
> **정체성 원칙 (회귀 금지)**: 이 게임은 관전형(attract) 게임이다. (1) 페이지 로드 즉시 오토 헌터가 플레이 시작 — 시작 화면 게이트 금지, (2) 오토 헌터 AI가 기본이며 유능하게 보여야 함, (3) 수동 개입은 "방향키/D-패드 탈취 + 터보 홀드" 수준의 저밀도 유지, 일정 시간 무입력 시 AI 복귀 유지, (4) 게임오버 7초 자동 재시작으로 방치 시 무한 순환.

### 1. 고정 타임스텝 게임 루프 (최우선)
- rAF 콜백에서 경과 시간을 누적해 60 updates/sec 고정 타임스텝으로 `update()` 실행, 탭 복귀 시 프레임당 최대 업데이트 수 제한(예: 5)
- 기존 프레임 단위 값(이동 4프레임/칸, 적 속도, 파티클 감쇠)은 60fps 기준이므로 값 변경 없이 동일 체감 유지. `Date.now()` 기반 점멸·자동 복귀 타이머도 틱 기준으로 통일

### 2. 반응형 캔버스·모바일 복구 (최우선)
- 캔버스에 `max-width: 100%` + 종횡비 유지 축소(내부 해상도 600×450 유지, CSS 스케일) — 375px에서 전장 전체가 보여야 함
- **D-패드 입력 경로 수정**: 터치/마우스 D-패드도 `disableAutoMode()`를 타도록 통일 — 수동 전환·5초 복귀 동작이 키보드와 동일해야 함
- D-패드·터보에 `preventDefault` 적용해 터치+마우스 중복 발화 제거, 짧은 화면에서는 컨트롤을 캔버스 하단에 항상 접근 가능하게 배치(스크롤 불가 전제이므로 뷰포트 내 수납)
- 캔버스 높이 산정은 `100dvh`/visualViewport 기준으로 헤더·컨트롤 공간을 제외해 계산

### 3. 피격 규칙 정리
- **점령지 위 안전 원칙**: 근접(1.5칸) 피격은 드로잉 중(트레일 위)에만 적용, 점령지·경계 위에서는 무적 — Qix 문법 복원. (경계 위협이 필요하면 별도 경계 순찰형 적(Sparx류) 추가로 해결, 근접 판정 재사용 금지)
- 스폰 후 2초 **실제 무적** 부여(기존 점멸 연출과 일치), 피격 처리를 프레임당 1회로 클램프
- `lastSafe`를 점령지 위를 이동할 때마다 갱신 — 피격 시 근처에서 리스폰
- 드로잉 중 역방향 입력 차단(자기 트레일 즉사 방지)

### 4. 적 반사 물리 수정
- 축별 침투 판정으로 분리: X 이동 후 충돌 시 vx만, Y 이동 후 충돌 시 vy만 반전 — 점령지 벽을 따라 미끄러지는 자연스러운 반사
- `±v×2` 점프 보정 제거, 침투량 기반 위치 되돌림으로 코너 끼임 방지

### 5. 레벨·난이도 곡선
- 80% 달성 시 게임 종료 대신 **다음 레벨로 연속 진행**: 필드 리셋 + 적 수 +1(상한 6), 적 속도 소폭 상승, 레벨 표시("ZONE N"). 승리 팡파르·통계는 레벨 전환 배너로
- 적 전멸 시(폐곡선 처치) 다음 웨이브 적을 수 초 내 투입 — 위협 0 상태로 방치되지 않게
- 목표 곡선: 오토 헌터 방치 시 레벨이 오르다 **수 분 내 목숨 소진 → 7초 재시작** 순환. 즉시 지거나 무한 생존하면 안 됨 — 다회 측정으로 검증

### 6. 점수·기록
- 점령 면적 점수화: 캡처 셀 수 × 기본 점수, 한 번에 큰 면적일수록 배율 상승(예: 5%+ 캡처 보너스), 적 근접 상태 캡처에 위험 보너스 — "과감한 컷"이 고득점이 되게
- localStorage에 최고 점수·최고 레벨 저장, HUD·게임오버 화면에 표시(신기록 강조), 게임오버 요약(최종 점령률/레벨/처치)

### 7. 오토 헌터 지능화 (attract 품질)
- 적이 멀 때는 4~8스텝 돌출 대신 **더 큰 직사각형 컷**(적과의 거리에 비례한 깊이) 시도 — 관전 시 영토가 시원시원하게 늘어나야 함
- 귀환 거리가 멀고 적이 접근 중이면 AI도 터보 사용(터보 연출 공유)
- AI 자동 복귀(5초 무입력)는 플레이어가 드로잉 중이 아닐 때만 발동 — 드로잉 중이면 트레일 완성 또는 피격 시점까지 유예
- HUD에 AI 상태 문구 노출(예: "HUNTING", "PANIC RETURN", "CUTTING 8×12") — AI의 판단이 보이게

### 8. Tailwind Play CDN 제거 → 정적 CSS
- Play CDN `<script>` 제거, 사용된 유틸리티만 수작업 CSS로 인라인 — 렌더 결과 시각적 동일 유지. Google Fonts는 유지 허용, 오프라인에서 폰트 폴백으로 정상 동작

### 9. 잔버그·메타데이터
- `endGame()` 진입 시 카운트다운 문구·타이머 초기화(“Restarting in 7s...”부터 시작)
- `AudioContext`를 첫 사용자 제스처 시 지연 생성
- `<meta name="description">`, Open Graph 태그(title/description/type), 인라인 SVG 파비콘 추가

## 검증 체크리스트
- [ ] 60Hz/144Hz에서 플레이어·적 이동 속도 동일 (고정 타임스텝, 시간 축 단일화)
- [ ] 375px 화면에서 전장 전체 표시, D-패드로 자동 해제→조작→5초 후 AI 복귀가 키보드와 동일하게 동작, 터치 중복 발화 없음
- [ ] 점령지 위 정지 상태에서 적이 스쳐도 피격 없음, 스폰 후 2초 무적(점멸과 일치), 한 프레임 다중 피격 없음
- [ ] 드로잉 중 역방향 입력이 무시됨(자기 트레일 즉사 없음), 피격 시 근처 안전 지점 리스폰
- [ ] 적이 점령지 벽에서 축별로 자연스럽게 반사(대각 되튕김·코너 끼임 없음)
- [ ] 80% 달성 시 다음 레벨 연속 진행(적 증가·가속), 적 전멸 시 새 웨이브 투입, 방치 시 수 분 내 게임오버 → 7초 재시작 순환 (다회 측정)
