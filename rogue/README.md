# Eternal Invasion: Roguelike Dungeon Edition

단일 HTML 파일(약 1200줄)로 만든 브라우저 로그라이크 던전 게임이다. 플레이어 캐릭터("Sopranian" 연주자 6종)와 NPC(적, [pizza](../pizza) 프로젝트의 손님 스프라이트 재사용)는 모두 이미지 에셋(`assets/sopranian/`, `assets/undeath/`)으로 그려지고, 아이템은 이모지, 효과음은 Web Audio API로 실시간 합성한다.

* https://kitscript.com/game
* https://github.com/textcube/kitscript/
* https://youtube.com/@textcube

## 게임 개요

연주자(Sopranian) 한 명이 언데드로 가득한 8×8 던전을 내려가는 게임이다. 목표는 각 층에서 열쇠를 가진 적을 처치해 열쇠를 얻고, 포탈을 통해 다음 층으로 내려가 최대한 깊은 층(Depth)에 도달하는 것이다.

게임을 시작하면 **연주자 선택 화면**이 뜬다. 소프라노/피아니스트/첼리스트/플루티스트/트럼페터/하피스트 6종 중 하나를 고르며, 각각 최대 MP·오토 이동 속도·데미지 감소·회피율 같은 고유 특성을 갖는다. 화면이 뜨자마자 하이라이트가 카드를 순서대로 한 칸씩 훑기 시작해 대기 시간이 멈춰 있는 화면으로 보이지 않는다. 카드를 직접 클릭하면 그 순간 바로 해당 연주자로 시작한다. 5초 안에 고르지 않거나 "START RANDOM"을 누르면, 그 순서대로 훑던 하이라이트가 이어서 점점 느려지는 **룰렛 연출**로 넘어가고, 무작위로 정해진 연주자 카드에 "SELECTED!" 표시가 뜬 뒤 곧바로 게임이 시작된다. 스핀 도중에도 카드를 클릭하면 그 선택이 즉시 우선한다.

기본은 **자동 진행(오토 플레이)** 방식이다. 1초마다 BFS 경로 탐색이 실행되어 연주자가 스스로 목표를 찾아 이동하고 전투한다. 방향키를 누르면 즉시 수동 조작으로 전환되고, 7초간 입력이 없으면 다시 오토 모드로 돌아간다. 상단 바의 **AUTO 버튼**을 눌러도 오토/수동을 바로 전환할 수 있으며, 오토 상태일 때는 금색으로 켜진다. 스페이스바로 일시정지/재개도 가능하다.

던전 그리드의 **좌측 상단 2×3칸은 플레이어 프로필 패널(♪ YOU), 우측 하단 2×3칸은 전투 상대(NPC) 프로필 패널(☠ TARGET)로 고정**되어 있다. 두 영역 모두 이동 불가능한 벽으로 취급되며(적·아이템·포탈도 생성되지 않는다), 보라색/붉은색으로 톤을 달리한 카드형 배경과 배너 라벨을 둘러 던전 타일과 한눈에 구분된다. 그 자리에는 캐릭터를 프레임 밖으로 살짝 넘치도록 크게 그린 초상화가 항상 떠 있어, 모바일처럼 타일이 작아지는 환경에서도 플레이어와 전투 상대를 또렷하게 볼 수 있다.

## 게임 규칙

- **마나(MP)가 곧 생명**이다. 최대 MP와 전투 특성은 선택한 연주자에 따라 다르며(예: 첼리스트는 MP 125에 피해 −2, 하피스트는 승리 시 MP +2 회복), 적과 전투할 때마다 적의 공격력만큼 MP가 깎인다. MP가 0이 되면 게임 오버.
- 적은 4종(좀비, 스켈레톤, 고스트, 드라큘라)이며 체력은 모두 1이라 한 번에 처치된다. 대신 **같은 종류의 적이 상하좌우로 붙어 있으면 클러스터**가 되어, 공격력이 `기본 공격력 + (클러스터 크기 − 1)`로 강해진다. 클러스터 하나를 공격하면 묶인 적 전체가 한꺼번에 제거된다.
- 층이 깊어질수록 적의 수(`12 + 층수×2`, 최대 32마리)와 기본 공격력이 함께 증가한다.
- 적 처치 시 28% 확률로 아이템이 드랍된다.

| 아이템 | 효과 |
|---|---|
| 🧪 포션 | MP 10 회복 |
| 💖 맥스MP | 최대 MP +10 (상한 200), 현재 MP도 +10 |
| 🔮 실드 | 4턴간 무적 |
| ⚡ 브레이커 | 6턴간 적의 클러스터 보너스 무시 (기본 공격력만 적용) |
| 🔑 열쇠 | 포탈 개방 — 층마다 무작위 적 1마리가 소지 |

- 열쇠를 얻으면 화면이 흔들리며 좌우 벽의 잠긴 포탈이 열리고, 포탈에 진입하면 다음 층으로 내려간다.
- 층은 생성될 때마다 "시작 지점에서 갈 수 있는 열쇠/처치 가능한 적/포탈"이 하나라도 있는지 검증하고, 없으면 다시 생성한다(최대 30회 재시도).
- 그래도 이길 수 있는 적이 없어 경로가 막히면 STUCK 판정이 뜨고 7초 카운트다운 후 **같은 층을 재생성**한다. 재시도(리롤)가 3회를 넘기거나 MP가 1 이하로 떨어지면 그때는 1층부터 다시 시작한다.
- 최고 도달 층수는 `localStorage`에 저장되어 상단 바의 BEST 배지와 게임오버 화면에 표시된다.

## 기술 구성

- **그리드**: 8×8 던전 타일 + 좌측 상단/우측 하단에 고정된 2×3 프로필 패널 2개. `isReserved(x, y)`가 두 패널의 좌표 범위를 판정하며, 층 생성 시 해당 칸은 처음부터 `type: 'reserved'`로 채워져 적·아이템·포탈 배치와 BFS 이동 후보에서 자동으로 제외된다.
- **연주자 선택**: `PLAYER_TYPES` 배열에 6종 연주자의 스탯(최대 MP, 피해 감소, 회피율, 오토 이동 딜레이 등)과 초상화 색상을 정의해두고, 시작 시 `#player-select` 오버레이에서 고르면 `makePlayerState(type)`이 그 스탯으로 `gameState.player`를 만든다. 카드 클릭은 즉시 확정되지만, 화면이 뜨는 즉시 `startIdleScan()`이 하이라이트를 카드 순서대로(`idleScanIndex`가 0→1→…→5→0으로 순환) 480ms 간격으로 옮겨 대기 시간이 정지 화면으로 보이지 않게 한다. 5초 대기 또는 "START RANDOM" 클릭 시 `spinToPlayer()`가 idle scan이 멈춰 있던 위치(`idleScanIndex`)에서 그대로 이어받아 감속 스핀으로 넘어간다 — 시작 위치와 목표 위치 차이(`baseOffset`)에 4바퀴를 더한 만큼(`baseOffset + 24`) 스텝을 반복하도록 미리 계산해두기 때문에 스핀이 항상 미리 뽑아둔 무작위 결과 위에서 정확히 멈춘다. 각 스텝 간 지연은 `55ms`에서 `315ms`까지 3차 함수로 늘어나 슬롯머신처럼 감속하고, 멈추면 `.selected` 클래스와 "SELECTED!" 배지가 뜬 뒤 800ms 후 게임을 시작한다. idle scan이든 스핀이든 도중에 카드를 직접 클릭하면 모든 타이머를 취소하고 그 선택을 즉시 우선시킨다.
- **프로필 패널**: 두 패널 모두 CSS Grid의 `grid-column`/`grid-row` span으로 배치된 카드이며, `data-label`(♪ YOU / ☠ TARGET)을 상단에 배너처럼 띄운다. 배경은 플레이어=보라, NPC=붉은 톤의 그라디언트로 서로 다르게 칠해 던전 타일(중립 다크 + 대각선 텍스처)과 뚜렷이 구분되도록 했다. 초상화 박스는 패널의 폭/높이를 각각 독립 비율(96%/84%)로 잡아 세로로 긴 패널 모양에 맞췄고, 내부 이미지는 그보다 살짝 크게(148~158%) 확대해 꽉 찬 느낌을 내되 `.profile-portrait`에 `overflow: hidden`을 걸어 카드 경계에서 정확히 크롭되므로 캐릭터가 패널 밖으로 튀어나오지 않는다. 플레이어(좌측 상단) 패널은 항상 `gameState.player`와 선택된 연주자 정보를 반영하고, NPC(우측 하단) 패널은 `gameState.lastEnemyProfile`(트레잇·공격력·클러스터·애니메이션 스냅샷)을 반영해 **직전에 싸운 적을 전투가 끝난 뒤에도 계속 표시**한다.
- **렌더링**: 던전 타일 DOM 요소는 층이 바뀔 때만 새로 만들고, 매 렌더에서는 내용이 실제로 바뀐 타일만 `innerHTML`을 갱신하는 diff 방식이라 변경되지 않은 유닛의 idle 애니메이션이 끊기지 않는다. 프로필 패널도 같은 방식으로 시그니처가 바뀔 때만 다시 그린다.
- **캐릭터 크기**: `.char-container`가 고정 픽셀이 아니라 타일의 92%(`width/height: 92%`)로 스케일링되어, `--tile-size`가 커지거나 작아질 때 항상 비례해서 커진다. 아이템 오브젝트(`.item-orb`)도 타일의 66%로 같은 방식으로 스케일링된다.
- **캐릭터 아트**: 플레이어는 `assets/sopranian/{id}.png`(연주자별 PNG 일러스트), NPC(적)는 [pizza/assets/customers](../pizza/assets/customers)에서 복사해온 `assets/undeath/{trait}_{pose}.svg`를 `<img>`로 로드해 `.char-container`에 얹는다. idle 부유·피격 플래시 같은 컨테이너 애니메이션은 그대로 적용된다. `getEnemySVG(type, stateClass)`가 idle이면 `waiting`, 전투 클래시 프레임(`ani-damage`)이면 `angry` 포즈를 선택한다.
- **타일 배경**: 적 타일은 트레잇별로 진하게 칠하는 대신 중립적인 어두운 카드 + 트레잇 색상의 은은한 링/글로우(`box-shadow`) + 대각선 텍스처만 두어, 스프라이트 자체의 색감이 배경과 부딪히지 않고 또렷하게 드러나도록 했다.
- **AI**: BFS 경로 탐색. 우선순위는 `바닥의 열쇠 → (열쇠 없음) 이길 수 있는 적 중 클러스터가 큰 것 → (열쇠 보유) 포탈` 순이다. `canKill()`이 현재 MP와 적 공격력을 비교해 이길 수 있는 적만 경로에 포함한다.
- **입력**: 방향키로 수동 이동(`handleInput`), 스페이스바로 일시정지, 상단 바 `AUTO` 버튼으로 오토/수동 즉시 전환. 7초간 조작이 없으면 자동으로 오토 플레이로 복귀한다.
- **사운드**: Web Audio API 오실레이터로 이동, 타격, 획득, 포탈 등의 효과음을 실시간 합성한다. 기본은 음소거이며 상단 바 버튼으로 켠다.
- **연출**: 화면 흔들림, 피격 비네트(붉은 테두리 플래시), HUD 쉐이크, 플로팅 데미지 텍스트(대상 타일의 실제 위치를 `getBoundingClientRect()`로 읽어 배치), 층 전환 커튼("DESCENDING") 등. 공격 애니메이션은 `--atk-dx`/`--atk-dy` CSS 변수(실제로 공격한 적과의 상대 좌표)를 `wizard-attack` 키프레임에서 읽어, 항상 오른쪽으로만 돌진하던 것과 달리 위/아래/좌/우 어느 방향의 적을 쳐도 그 방향으로 돌진한다. 충돌 순간에는 대상 타일 위에 X자 슬래시 플래시(`.impact-fx`)와 게임 영역 전체의 미세한 줌 펄스가 함께 재생된다. 클러스터가 처치되면 즉시 사라지는 대신 `assets/undeath/{trait}_leaving.svg` 포즈로 0.32초간 축소·회전·페이드된 뒤 비워진다.
- **전투 가독성**: 플레이어 타일은 상시 breathing glow(`player-pulse`)와 살짝 확대된 크기로 보드에서 항상 눈에 띄고, `canKill()` 결과가 `false`인 적 타일은 채도/밝기를 낮추고 붉은 경고 테두리 + ⚠ 아이콘을 얹어 "지금은 이길 수 없는 상대"임을 관전 중에도 바로 알 수 있다.
- **HUD MP 게이지**: 상단 바의 MP 배지는 숫자 표기 위에 `#mp-fill` 막대가 겹쳐 있어(`width`를 MP 비율로 매 렌더마다 갱신) 잔량을 한눈에 볼 수 있는 게이지 역할도 겸한다.
- **반응형**: `--tile-size`가 `clamp(36px, 10vw, 96px)`라 모바일에서도 타일과 캐릭터가 충분히 크고, `body`는 세로 스크롤이 가능하다. 프로필 패널의 연주자 특성 텍스트(예: "-2 enemy ATK")는 480px 이하 폭에서 숨겨져(`.profile-note`) 좁은 화면에서 초상화를 가리지 않는다.
- **상단 바**: 패딩·테두리·그림자를 키운 카드형 디자인이며, 아이콘 버튼(홈/유튜브/사운드)은 각각 다른 hover 색 대신 브랜드의 청록색(`--accent-mana`) 하나로 통일해 단순하게 정리했다. `#top-bar`는 `flex-wrap`을 켜두고 480px 이하에서 패딩·아이콘·배지 크기를 줄이는 미디어쿼리를 둬서, 브랜드/스탯/아이콘 세 그룹이 좁은 화면에서 두 줄로 자연스럽게 줄바꿈되고 넘치지 않는다. 이 작업 중 전역 `box-sizing: border-box`가 빠져 있던 것을 발견해 함께 추가했다(패딩이 있는 `width:100%` 요소가 좁은 화면에서 넘치는 근본 원인).

## 개발 이력

**프로필 초상화 상하 여백 불균형 수정 (2차)** — 박스 자체의 `margin-top` 잔재를 걷어낸 뒤에도(1차 수정) 캐릭터가 여전히 위로 치우쳐 보인다는 스크린샷을 받았다. `assets/sopranian/flutist.png`를 캔버스에 그려 알파 채널을 픽셀 단위로 스캔해보니 원본 이미지 속 캐릭터는 이미 거의 중앙(상단 여백 4.7%, 하단 여백 5.5%)에 위치해 있었다 — 그런데 `#player-profile-panel .profile-portrait img`/`#enemy-profile-panel .profile-portrait img`에 남아있던 `transform: translateY(5%)`(`4%`)가 이미 중앙에 있는 이미지를 추가로 아래로 밀어 다시 불균형을 만들고 있었다. 두 `translateY()`를 제거해 `object-fit: contain`이 원본 그대로 중앙 정렬하도록 되돌렸다. 계측 결과 이미지 박스의 상하 오버플로우 여백 차이가 2px 이내로 좁혀졌다.

**프로필 초상화 상하 여백 불균형 수정 (1차)** — YOU/TARGET 패널의 초상화 위쪽에는 여백이 크고 아래쪽은 거의 없어 보이는 문제가 있었다. 원인은 기본 `.profile-portrait` 규칙의 `margin-top: 16px`가 `#player-profile-panel`/`#enemy-profile-panel`용 더 구체적인 규칙에서 재선언되지 않아 그대로 남아있었기 때문— CSS는 규칙 전체가 아니라 프로퍼티 단위로 캐스케이드되므로, 특정 규칙이 `margin-top`을 다시 쓰지 않으면 기본값이 계속 적용된다. 두 패널 모두 `margin-top: 0`을 명시해 flex `justify-content: center`가 순수하게 대칭으로 정렬하도록 고쳤다(계측 결과 상하 여백 차이가 22px에서 2px로 줄었고, 남은 2px는 `.profile-panel` 패딩(`10px`/`8px`) 차이에 따른 의도된 값이다).

**선택 대기 중 정지 화면 제거** — 연주자 선택 화면이 뜬 뒤 5초 대기 동안 룰렛 스핀이 시작되기 전까지는 카드들이 완전히 정지해 있어 화면이 멈춘 것처럼 보였다. `startIdleScan()`을 추가해 화면이 뜨는 즉시 하이라이트가 카드를 순서대로(왼쪽→오른쪽, 줄바꿈 포함) 계속 옮겨 다니게 했고, 실제 무작위 선택이 시작되는 시점(5초 경과 또는 START RANDOM 클릭)에는 `spinToPlayer()`가 idle scan이 멈춘 위치에서 그대로 이어받아 감속 스핀으로 자연스럽게 전환되도록 했다.

**타격 이펙트 크기 버그 수정** — 전투 클래시 순간의 X자 슬래시(`.impact-fx`)가 이따금 화면에 꽉 찰 정도로 거대하게 나타나는 버그를 발견했다. 원인은 `width: 60%; height: 60%;`가 `#game-area`(던전 보드 전체)에 `position: absolute`로 붙은 요소라 타일이 아니라 보드 전체 크기의 60%로 계산되던 것이었다. `calc(var(--tile-size) * 0.6)`으로 바꿔 타일 크기에 비례하도록 고쳤다.

**연주자 선택 룰렛 연출 추가** — 무작위 선택(5초 대기 또는 START RANDOM 클릭)이 즉시 결과만 보여주던 것을, 하이라이트가 카드들을 좌→우로 훑으며 점점 느려지는 룰렛 스핀으로 바꿨다. 미리 뽑아둔 결과 위에 정확히 멈추도록 스텝 수를 역산해두고, 멈추면 "SELECTED!" 배지와 안내 문구를 띄운 뒤 게임을 시작한다. 스핀 중에도 카드를 직접 클릭하면 그 선택이 즉시 우선한다.

**아이콘을 이모지에서 SVG로 교체** — 홈/유튜브/음소거 버튼의 색을 바꿔도 실제로는 그대로처럼 보인다는 지적을 받았다. 원인은 🏠·📺·🔇 같은 이모지 글리프가 폰트에 고유 색이 박혀 있어 CSS `color`를 아예 무시하기 때문이었다. 세 아이콘 모두 `stroke="currentColor"`를 쓰는 인라인 SVG(집 모양, 원 안 재생 삼각형, 스피커)로 교체해 버튼의 기본/hover 색이 실제로 아이콘에 반영되도록 했다. 음소거 토글은 `innerText` 이모지 교체 대신 두 SVG 문자열(`ICON_SOUND_ON`/`ICON_SOUND_OFF`)을 `innerHTML`로 바꿔치기하는 방식으로 바뀌었다.

**AUTO 토글 버튼 추가** — 참고 이미지(음소거/AUTO/1x/홈/재생 버튼 바)의 스타일을 반영해, 기존 아이콘 버튼을 모두 둥근 사각 필(pill) 모양으로 다듬고 `AUTO` 텍스트 버튼을 새로 추가했다. `gameState.isAuto`가 참이면 금색으로 켜지고, 클릭하면 `handleInput`의 수동 전환과 같은 방식(오토/수동 토글 + `lastInputTime` 갱신)으로 즉시 전환된다. `.icon-btn`을 고정 `width`에서 `min-width`로 바꿔 아이콘 전용 버튼은 정사각형을 유지하면서 텍스트가 있는 AUTO 버튼만 내용에 맞게 넓어지도록 했다.

**상단 바 두께감 + 아이콘 색상 단순화** — 브랜드/상태 바(`#top-bar`)를 패딩·테두리·그림자를 키운 카드형으로 바꾸고, 아이콘 버튼(🏠/📺/🔇)의 hover 색을 각각 청록/빨강/금색으로 흩어져 있던 것에서 청록(`--accent-mana`) 하나로 통일했다. 이 변경으로 좁은 화면에서 상단 바가 넘치는 버그가 드러나(원인: 전역 `box-sizing` 미설정 + 480px 이하 대응 부재) `box-sizing: border-box` 전역 리셋과 `#top-bar`의 `flex-wrap` + 모바일 축소 미디어쿼리를 함께 추가해 해결했다.

**[DESIGN_IMPROVEMENTS.md](DESIGN_IMPROVEMENTS.md) 우선 적용 6종** — 문서의 "우선 적용 추천 순서" 표에 있던 항목을 순서대로 구현했다: ① 플레이어 타일 상시 pulse 글로우로 보드 위 위치 강조, ② `canKill()` 기반으로 처치 불가능한 적을 채도 저하 + 붉은 경고 테두리/⚠ 아이콘으로 구분, ③ 충돌 순간 X자 슬래시 플래시(`.impact-fx`) + 화면 미세 줌, ④ 480px 이하에서 프로필 패널의 연주자 특성 텍스트를 숨겨 모바일에서 초상화 가림 방지, ⑤ 그동안 미사용이던 `*_leaving.svg` 포즈를 적 처치 시 축소·회전·페이드 연출로 활용, ⑥ MP 스탯 배지를 텍스트 위에 겹친 fill 막대로 바꿔 게이지처럼 보이게 함.

**프로필 패널 상단 공백 축소** — 2×3 패널이 폭보다 세로로 긴데 초상화를 `aspect-ratio: 1/1`(폭 기준 정사각형)로 잡아서 라벨 아래에 빈 공간이 크게 남았다. 폭/높이를 각각 독립적인 퍼센트(96%/84%)로 지정해 정사각형 제약을 없애, 초상화가 패널의 세로 공간을 훨씬 더 채우도록 했다.

**프로필 초상화 크롭** — 확대된 캐릭터 이미지가 패널 밖 던전 타일 위로 그대로 삐져나오던 것을, `.profile-portrait`의 `overflow: visible`을 `hidden`으로 바꿔 카드 경계에서 깔끔하게 잘리도록 했다(경계 자체에 살짝 라운드 처리도 추가).

**공격 방향 반영 + 프로필 텍스트 경계 고정** — 마법사(연주자)의 공격 돌진 애니메이션이 실제 공격 방향과 무관하게 항상 오른쪽으로만 재생되던 것을, 공격 대상과의 상대 좌표(`--atk-dx`/`--atk-dy`)를 반영해 위/아래/좌/우 어느 적을 쳐도 그 방향으로 돌진하도록 고쳤다. 또한 프로필 패널의 초상화를 프레임 밖으로 크게 오버플로우시키면서 이름/스탯 텍스트가 카드 경계를 벗어나거나 그림 아래 묻히던 문제를, 텍스트를 `.profile-stats`로 묶어 카드 하단에 절대 위치로 고정하고 캐릭터 이미지보다 위 레이어(z-index)에 배치해 해결했다.

**패널·타일 배경 분리 + NPC 초상화 확대** — 다른 세션에서 플레이어 캐릭터를 "Sopranian" 연주자 6종 선택 시스템(`assets/sopranian/`)으로 교체하는 큰 변경이 들어온 뒤, 플레이어 패널 초상화만 프레임 밖으로 크게 오버플로우되고 NPC(TARGET) 패널은 예전 크기 그대로라 균형이 안 맞았다. NPC 패널에도 동일한 오버플로우 확대를 대칭 적용했고, 두 프로필 패널의 배경을 각각 보라/붉은 톤 그라디언트 카드 + 상단 배너 라벨(♪ YOU / ☠ TARGET)로 다시 디자인해 이동 가능한 던전 타일(중립 다크 + 은은한 대각선 텍스처, 그리드 전체에 비네트 추가)과 한눈에 구분되도록 했다.

**캐릭터 확대 + 그래픽 개선** — 캐릭터가 타일에 비해 작아 보인다는 피드백에 따라, `.char-container`를 고정 48px에서 타일의 92%로 바꿔 사실상 약 2배 더 크게 보이도록 했다(타일 크기 자체도 `clamp(36px, 10vw, 96px)`로 소폭 키움). 적 타일 배경은 트레잇별 진한 단색 대신 중립 카드 + 은은한 컬러 글로우 링으로 바꿔 새 스프라이트 색감과 충돌하지 않게 했고, 프로필 패널은 `YOU`/`TARGET` 라벨과 훨씬 큰 초상화(패널 폭의 68%)로 다시 디자인했다. 아이템 오브·잠금 포탈 아이콘·유닛 스탯 텍스트도 타일 크기에 비례하도록 정리했다.

**NPC 아트 교체 + 패널 위치 반전** — 손그림 SVG로 그려지던 6종 트레잇(좀비/스켈레톤/레이스/구울/리치/데스나이트)을 걷어내고, [pizza](../pizza) 프로젝트의 손님 스프라이트(좀비/스켈레톤/고스트/드라큘라, 각 waiting·angry·leaving 포즈)를 `rogue/assets/undeath/`에 복사해 그대로 재사용하도록 트레잇을 4종으로 교체했다. 플레이어 프로필 패널은 좌측 상단, NPC 프로필 패널은 우측 하단으로 위치를 맞바꿨다.

**레이아웃 개편** — 던전을 10×10에서 8×8로 줄이고, 이동 불가능한 2×3 프로필 패널 2개를 고정 배치했다(이후 좌측 상단=플레이어/우측 하단=NPC로 위치 확정). 플레이어 패널은 항상 마법사 큰 초상화 + MP를, 적 패널은 직전 전투 상대의 큰 초상화 + ATK를 보여준다. 타일 크기도 처음으로 뷰포트 기반 `clamp()`로 바꿔 모바일에서도 캐릭터가 잘 보이도록 했다.

**버그 수정** — 정의되지 않은 `handleInput` 호출로 인한 방향키 ReferenceError, 적 배치 수 무상한으로 인한 깊은 층 무한 루프, 매초 전체 리렌더로 인한 애니메이션 리셋, 포탈 SVG 그라디언트 ID 중복, 플로팅 텍스트 좌표 매직 넘버, 죽은 `paused` 상태 변수를 모두 고쳤다.

**그 외 개선** — 로그 패널에 전투/아이템/층 하강 이벤트 기록, `localStorage` 기반 최고 도달 층수(BEST) 표시, 내부 상태명을 `hp`/`maxHp`에서 `mp`/`maxMp`로 통일(HUD의 "MP" 표기와 일치), STUCK 시 1층 리셋 대신 같은 층 리롤(3회 실패 시에만 전체 리셋), 층 생성 시 도달 가능한 목표 보장(`findPath()` 검증 후 최대 30회 재생성).

## 남은 개선 여지

디자인·UI·그래픽 개선 제안은 [DESIGN_IMPROVEMENTS.md](DESIGN_IMPROVEMENTS.md)에 우선순위와 구현 힌트를 붙여 별도로 정리해두었다.

- 여전히 단일 파일 구조다. 기능을 더 붙일 계획이라면 CSS/JS 분리와 렌더링·게임 로직·오디오 모듈 분리를 고려할 만하다.
- 수동 조작은 방향키만 지원한다. 터치 기기에서 직접 조작하려면 온스크린 방향 버튼이 필요하다.
- 적 프로필 패널은 "직전 전투 상대"만 기억한다. 여러 마리와 연속으로 싸운 로그를 남기고 싶다면 작은 히스토리 목록을 추가할 수 있다.
- 복사해온 스프라이트 중 `zombie_satisfied` 포즈는 아직 쓰이지 않는다(`leaving`은 적 처치 연출에 활용 중).
- pizza 프로젝트의 `assets/undeath/`(프랑켄슈타인·마녀·미이라·호박·임프)에도 같은 스타일의 손님 스프라이트가 더 있다. 트레잇을 늘리고 싶다면 이 폴더도 복사해 재사용할 수 있다.

## 실행 방법

빌드 없이 `index.html`을 브라우저에서 열면 바로 실행된다. 로컬에서 미리보기 도구로 확인할 때는 `.claude/launch.json`에 등록된 `rogue-static`(Python `http.server`, 포트 8791) 설정을 사용했다. 사운드는 상단 바의 🔇 버튼을 눌러 활성화한다.
