# 디자인 · UI · 그래픽 개선안

`index.html` 현재 버전(8×8 그리드 + 코너 프로필 패널 + Sopranian 연주자/pizza 언데드 스프라이트)의 스크린샷을 기준으로 정리한 개선 제안 목록이다. 항목마다 우선순위(★ 높음 / ☆ 중간 / ○ 낮음)와 구현 힌트를 붙였다.

**우선 적용 6종(1-1, 1-2, 4-1, 7-1, 3-3, 5-1)은 구현 완료됨** — 각 항목 제목에 ✅ 표시. 나머지는 아직 미착수 제안이다.

---

## 1. 시각적 위계 (Visual Hierarchy)

### ★ 1-1. 플레이어 현재 위치가 잘 안 보인다 ✅ 완료
프로필 카드 속 HARPIST는 크고 화려한데, 정작 보드 위의 플레이어 토큰은 일반 적 타일과 크기·밝기가 비슷해서 "지금 어디 있는지"를 한 번에 찾기 어렵다.
- 플레이어 타일에 상시 펄스 글로우(`@keyframes` box-shadow 확장/수축)를 추가해 보드에서 유일하게 "숨쉬는" 타일로 만든다.
- 플레이어 타일 아래에 바닥 스포트라이트(radial-gradient 원형 발광)를 깔거나, 타일 z-index를 살려 살짝 확대(`transform: scale(1.08)`)한다.
- 프로필 카드와 보드 토큰을 가는 연결선 또는 동일한 테두리 색으로 시각적으로 묶어주면 "카드 = 이 토큰"이라는 연결이 즉시 읽힌다.
- **적용**: `player-pulse` 키프레임 + `transform: scale(1.07)` 상시 적용. 스포트라이트/연결선은 적용 안 함(과했다고 판단, 필요 시 추가 가능).

### ★ 1-2. 어떤 적을 이길 수 있는지 색으로 구분 ✅ 완료
현재 `canKill()` 정보가 UI에 전혀 드러나지 않아, 관전 중에는 AI가 왜 특정 적을 피하는지 알 수 없다.
- 처치 가능한 적: 현재 테두리 유지. 처치 불가능(공격력 ≥ 내 MP) 적: 채도를 낮추고(`filter: saturate(0.4) brightness(0.7)`) 붉은 경고 테두리.
- `updateTile()`의 enemy 분기에서 `canKill(tile)` 결과로 클래스 하나만 토글하면 된다. 시그니처(`tileSignature`)에 canKill 결과를 포함시켜야 MP 변화 시 갱신된다.
- **적용**: `.tile-enemy.too-strong`(saturate 0.35/brightness 0.65 + 붉은 테두리 `!important`) + `::after`로 ⚠ 아이콘. 강제로 MP를 낮춰 15개 적 전부 desaturate되는 것과 정상 복귀를 확인.

### ☆ 1-3. 클러스터 시각화 강화
지금은 같은 클러스터가 초록 테두리로만 묶여 보이는데, 인접 타일 사이 간격 때문에 "한 덩어리"라는 느낌이 약하다.
- 클러스터에 속한 타일들 사이의 gap을 가로지르는 연결 브릿지(가상요소로 4px 두께의 짧은 바)를 그리거나,
- 클러스터 전체에 같은 색조의 배경 틴트를 깔고 테두리는 외곽선만 남긴다(인접 타일끼리 맞닿는 변의 테두리 제거).

## 2. 프로필 패널

### ★ 2-1. TARGET 패널의 정보 밀도
스크린샷에서 GHOST 초상화 주변에 아직 여백이 있고, 표시 정보가 이름+ATK뿐이다. 공간을 더 쓸 수 있다:
- 처치 수 카운터: 이번 판에서 잡은 해당 트레잇 수 ("GHOST ×7 defeated").
- 위협도 게이지: `ATK / 내 MP` 비율을 작은 바로 표시 — 관전 긴장감이 생긴다.
- 전투 순간에는 패널 테두리를 번쩍이게(`hud-shake` 재활용) 해서 "지금 이 녀석과 싸우는 중"을 강조.

### ☆ 2-2. 패널 프레임 장식
카드 느낌을 더 살리려면 모서리 장식(coner ornament)이 효과적이다. 이미지 없이도 가능:
- `::before`/`::after` + `border-image` 또는 CSS `conic-gradient`로 모서리에 금속 프레임 느낌의 꺾쇠(⌜⌟) 장식.
- YOU 패널은 금색/보라, TARGET 패널은 붉은 톤으로 통일.

### ○ 2-3. 초상화 배경 무대 연출
연주자 콘셉트를 살려 YOU 패널 초상화 뒤에 은은한 무대 조명(위에서 내려오는 원뿔형 빛)을, TARGET 패널에는 묘지 안개(하단에서 올라오는 반투명 그라디언트)를 깔면 두 패널의 성격 대비가 강해진다.

## 3. 보드 · 타일 그래픽

### ★ 3-1. 빈 타일이 너무 균일하다
현재 빈 타일은 모두 동일한 대각선 텍스처라 바닥이 밋밋하다.
- 타일마다 미세하게 다른 밝기 변주를 주면(`tile-${x}-${y}` 좌표 기반으로 3~4단계 배경 클래스를 결정적으로 배정) 돌바닥 느낌이 난다. 랜덤이 아니라 좌표 해시 기반이면 리렌더에도 안정적이다.
- 아주 낮은 확률(5% 내외)로 장식 타일(균열, 이끼, 해골 문양 등 이모지/SVG 워터마크 10% 불투명도)을 섞는다.

### ☆ 3-2. 포탈 방향성
좌/우 벽의 포탈이 잠긴 상태에서는 자물쇠 아이콘만 보여서 "출구"라는 느낌이 약하다.
- 잠긴 포탈에도 비활성 소용돌이(회색, 저속 회전)를 깔고 자물쇠를 그 위에 겹치면 "잠긴 문"이라는 서사가 생긴다.
- 열리는 순간 기존 `portal-reveal` + 화면 흔들림에 더해, 포탈에서 보드 중앙으로 퍼지는 파동 링 1회를 추가하면 개방 이벤트가 확실히 각인된다.

### ☆ 3-3. 적 사망 연출 ✅ 완료
현재 적은 클러스터 단위로 즉시 사라진다. `assets/undeath/`에 이미 있는 `leaving` 포즈를 활용:
- 사망 시 `leaving` 스프라이트로 교체 + `playerEnterPortal`류 축소·회전·페이드 애니메이션 0.3초 → 그 후 타일 비움.
- `startBattleSequence()`의 클러스터 제거 직전에 포즈 교체 렌더 한 번과 `setTimeout` 300ms만 추가하면 된다.
- **적용**: `.ani-leaving`(scale 0.15 + rotate -18deg + fade, 0.32s) + `getEnemySVG`가 `ani-leaving` 상태에서 `leaving` 포즈를 그리도록 매핑. 클러스터 좌표 전체에 동시 적용 후 320ms 뒤 비움(개별 타일마다 순차 지연은 미적용 — 4-1의 "연쇄 소멸" 아이디어와 겹치는 후속 개선 여지).

### ○ 3-4. 아이템 오브 차별화
아이템 5종이 모두 같은 검은 원 안 이모지라 위계가 없다. 열쇠(진행 필수)만이라도 원형 오브 대신 육각형/방패형 프레임 + 강한 금색 발광으로 차별화하면 "이건 꼭 먹어야 하는 것"이 읽힌다.

## 4. 전투 연출

### ★ 4-1. 타격 이펙트 부재 ✅ 부분 완료
공격 시 플레이어가 돌진(방향 반영됨)하지만 충돌 순간의 임팩트가 없다.
- 충돌 프레임에 대상 타일 위 슬래시 스파크(CSS로 그린 X자 빛줄기 0.15초) 또는 화면 미세 줌(게임 영역 `scale(1.01)` 0.1초)을 추가.
- 클러스터 처치 시에는 제거되는 타일들이 중심에서 바깥 순서로 0.05초 간격 연쇄 소멸하면 훨씬 시원하다.
- **적용**: `.impact-fx`(X자 슬래시, 0.35s) + `#game-area.impact`(scale 1.015 줌, 0.15s)를 데미지 프레임에 추가. MutationObserver로 전투 중 실제 생성 확인. **미적용**: 클러스터 중심-바깥 순서 연쇄 소멸(3-3의 일괄 leaving 연출로 어느 정도 대체됨 — 순차 스태거는 여전히 남은 여지).

### ☆ 4-2. 플로팅 텍스트 스타일 위계
"PURGE", "-2", "KEY DROPPED"가 모두 같은 스타일이다. 데미지 숫자는 크고 두껍게+위로 팝, 상태 텍스트(BLOCK/EVADE)는 옆으로 슬라이드, 획득 텍스트(ITEM/KEY)는 반짝임 추가 — 세 가지 애니메이션 변형으로 나누면 정보가 즉시 구분된다.

### ○ 4-3. MP 위험 상태 연출
MP가 최대치의 25% 이하로 떨어지면: HUD MP 배지 상시 붉은 펄스, 화면 가장자리 얇은 비네트 상시 표시, 플레이어 카드 초상화에 흑백 필터 20%. "위험하다"는 감각을 시스템 전체가 말해주게 된다.

## 5. HUD · 로그

### ☆ 5-1. 상단 바 정보 그룹화 ✅ 완료 (게이지 부분만)
FL/BEST/MP/열쇠가 같은 무게로 나열되어 있다. MP는 생명이므로 숫자 대신(또는 병행) 게이지 바로 바꾸면 잔량이 직관적으로 보인다. `#hud-mana` 배지 안에 `linear-gradient` 배경 폭을 %로 조절하는 방식이면 DOM 추가 없이 가능하다.
- **적용**: `#hud-mana` 안에 `#mp-fill`(`position:absolute`, z-index 0) 막대를 추가하고 텍스트는 z-index 1로 위에 띄움. `render()`에서 MP 비율로 `width`를 매번 갱신. FL/BEST/열쇠의 그룹화 자체(시각적 무게 재배치)는 미적용.

### ☆ 5-2. 로그 패널 아이콘화
텍스트만 흐르는 로그에 이벤트별 아이콘 접두(⚔ 전투 / 🧪 아이템 / 🔑 열쇠 / ⬇ 층 이동)를 붙이고, 데미지 수치는 붉은색 하이라이트. `addLog(msg, type)`으로 시그니처만 확장하면 된다.

### ○ 5-3. 층 전환 커튼 개선
"DESCENDING" 커튼이 단색 배경이라 밋밋하다. 현재 층의 보드 스크린샷이 아래로 흘러가는 패럴랙스, 또는 계단 실루엣 SVG 하나만 깔아도 하강 서사가 생긴다.

## 6. 색·타이포그래피 시스템

### ☆ 6-1. 색 역할 정리
현재 보라(플레이어/포탈), 금색(열쇠/BEST/STUCK), 붉은색(적/MP/데미지), 청록(브랜드/버튼)이 혼용된다. 역할표를 정해 CSS 변수에 주석으로 못박아두면 앞으로의 수정에서 일관성이 유지된다:
- `--accent-wizard`(보라) = 플레이어·아군 / `--accent-red` = 위협·손실 / `--accent-gold` = 목표·보상 / `--accent-mana`(청록) = 시스템·내비게이션.

### ○ 6-2. 디스플레이 폰트 도입
전부 Segoe UI라 게임 타이틀/라벨의 개성이 약하다. 오프라인 단일 파일 정책을 유지하려면 시스템 폰트 스택으로 `Impact, 'Arial Black'`을 라벨(YOU/TARGET/GAME OVER)에만 써도 인상이 달라진다. 웹폰트를 허용한다면 Cinzel(중세 명조풍) 하나만 타이틀·라벨에 적용.

## 7. 반응형 · 접근성

### ★ 7-1. 모바일 프로필 패널 텍스트 겹침 ✅ 완료
375px에서 YOU 패널의 "PIANIST / Faster auto / MP 75/90" 3줄이 초상화를 상당히 가린다. 패널 폭이 특정 기준(예: 90px) 이하일 때는 이름+MP만 남기고 특성 줄을 숨기는 미디어쿼리(또는 컨테이너 쿼리)가 필요하다.
- **적용**: `@media (max-width: 480px) { .profile-note { display: none; } }`로 특성 줄만 숨김. 375px 모바일 스크린샷에서 이름+MP 2줄만 남는 것 확인.

### ☆ 7-2. 터치 조작
현재 모바일은 관전 전용이다. 보드 위 스와이프 → 방향 입력 매핑(touchstart/touchend 좌표 차이로 `handleInput` 호출)이면 온스크린 버튼 없이도 수동 조작이 가능해진다.

### ○ 7-3. 접근성 기본기
- `prefers-reduced-motion` 미디어쿼리에서 화면 흔들림·펄스류 애니메이션 비활성화.
- 적 타일에 `aria-label`("zombie, attack 3, cluster of 2") 부여.
- 승패 정보가 색에만 의존하지 않도록(1-2의 처치 불가 표시에 채도 외에 🚫 아이콘 병행).
