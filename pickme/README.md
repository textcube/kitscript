# PICK ME — Retro Stage Select

레트로 CRT 감성의 **랜덤 스테이지 선택기**입니다. 접속하면 kitscript의 게임/툴 목록이 카드 그리드로 표시되고, 룰렛처럼 카드를 순회하는 셔플 연출 후 가중치 기반으로 선택된 스테이지로 자동 이동합니다. 원하는 카드를 직접 클릭해 수동 선택할 수도 있고, STOP 버튼으로 자동 추첨을 중단할 수도 있습니다.

단일 파일(`index.html`)로 동작하며 외부 라이브러리 의존성이 없습니다.

## 동작 방식 분석

### 전체 흐름

```
페이지 로드
  → init(): 라우트 필터링 → 테마 적용 → 카드 렌더링 → 그리드 크기 계산
  → pickOne(): 가중치 추첨으로 당첨 라우트를 미리 결정
  → 600ms 후 startSequence(): 카드를 랜덤 순회하는 셔플 연출 (22~29 스텝, 점점 감속)
  → finalize(): 당첨 카드 강조(flash) → 1.2초 후 해당 URL로 window.location.replace()
```

셔플 연출은 눈속임이며, **당첨자는 연출 시작 전에 이미 확정**되어 있습니다. 마지막 스텝에서 당첨 카드에 강제로 커서를 맞춥니다.

### 데이터 구조

- `ROUTES` — 41개의 큐레이션된 라우트 배열. 각 항목은 `url`, `weight`(1~3, 추첨 확률 가중치), `theme`(city/space/tool/default), `label`, `image`(썸네일), `enabled`(노출 여부)를 가집니다.
- `ROOT_STAGE_DIRS` — 디렉터리 이름만 넣으면 `buildAutoRoutes()`가 테마/가중치/라벨을 자동 추론해 라우트를 생성하는 확장 통로. 현재는 비어 있습니다.
- `mergeRoutes()` — 큐레이션 목록과 자동 생성 목록을 URL 기준으로 중복 제거하며 병합(큐레이션이 우선).

### 주요 메커니즘

| 기능 | 구현 |
|---|---|
| 가중치 추첨 | `pickOne()` — 가중치 합 기준의 표준 roulette-wheel 방식 |
| 연속 중복 방지 | 직전 당첨 URL을 `localStorage("pickme_last_url")`에 저장하고 다음 추첨 풀에서 제외 |
| 테마 필터 | `?theme=city` 같은 쿼리 파라미터로 풀을 좁히고 액센트 컬러도 전환 |
| 수동 선택 | 그리드에 클릭 이벤트 위임 — 셔플 중에도 카드를 클릭하면 즉시 그 스테이지로 확정 |
| 자동 추첨 중단 | STOP 버튼 — 셔플 타이머를 멈추고 MANUAL MODE로 전환, RE-ROLL로 재추첨 가능 |
| 감속 연출 | `delay = 40 + progress^5 * 800` — 5제곱 이징으로 마지막에 급감속 |
| 반응형 그리드 | `updateGridMetrics()` — 카드 수와 뷰포트 크기로 열 수(2~6)와 카드 높이를 동적 계산 |
| 안전한 URL 해석 | `resolveSafeUrl()` — 상대 경로를 origin 기준 절대 경로로 변환 (`file://`, `blob:` 환경 방어) |

### 설계 의도

kitscript 게임들의 공통 정체성인 **관전형(attract-first)** 패턴을 따릅니다. 사용자가 아무것도 하지 않아도 추첨이 자동 진행되고, 개입(카드 클릭, STOP)은 선택 사항입니다. STOP 버튼도 자동 진행을 막는 게 아니라 "구경하다가 마음이 바뀌면 개입"하는 저밀도 조작으로 설계했습니다.

## 개선점

### 반영됨

- **랜덤 선택 중단 버튼** — 셔플 중 `■ STOP`을 누르면 자동 추첨이 멈추고 MANUAL MODE가 되어 원하는 카드를 천천히 고를 수 있습니다. 버튼은 `▶ RE-ROLL`로 바뀌어 다시 누르면 새로 추첨합니다. 선택이 확정되면 버튼은 숨겨집니다.

### 제안 (미반영)

1. **확정 후 취소 불가** — `finalize()` 이후 1.2초 내에 이동을 취소할 방법이 없습니다. 확정 화면에 짧은 CANCEL 유예를 주면 오클릭 구제가 됩니다.
2. **접근성** — 카드가 `div`라 키보드 포커스/Enter 선택이 불가능합니다. `role="button"` + `tabindex="0"` 또는 실제 `<button>` 전환이 필요합니다. 또한 당첨 카드의 `flash` 애니메이션(0.1s 무한 반전)은 광과민성 사용자에게 위험할 수 있어 `prefers-reduced-motion` 대응을 권장합니다.
3. **외부 이미지 의존** — 카드 썸네일이 `kitscript.com`과 Unsplash의 원격 이미지 41장에 의존합니다. 오프라인/로컬 실행 시 카드가 빈 배경이 되고, 첫 로드 트래픽도 큽니다. 로컬 에셋화 또는 `loading="lazy"` 배경 지연 로드를 고려할 만합니다.
4. **데드 코드** — `ROOT_STAGE_DIRS`가 빈 배열이라 `buildAutoRoutes()`/`inferTheme()`/`inferWeight()`/`slugToLabel()`과 `SPACE_DIRS`/`CITY_DIRS`/`TOOL_DIRS` 세트가 현재 실행 경로에서 쓰이지 않습니다. 자동 등록 기능을 살릴 계획이 없다면 제거해 파일을 줄일 수 있습니다.
5. **데이터 중복** — 라우트가 41개로 늘면서 인라인 배열이 파일의 1/3을 차지합니다. `routes.json` 분리 또는 게임 허브와의 목록 공유를 고려할 만합니다. 일부 이미지(`r-type`, `crystalis`, `metal-warriors`)는 두 라우트에서 중복 사용 중입니다.
6. **weight 노출 방식** — 카드의 `LV.n` 표기는 추첨 가중치인데, 사용자에게는 난이도처럼 읽힐 수 있습니다. `x3` 같은 확률 표기나 툴팁 설명이 더 명확합니다.
7. **테마 파라미터 검증 없음** — `?theme=foo`처럼 존재하지 않는 테마를 주면 풀이 비어 `/`로 리다이렉트됩니다. 알 수 없는 테마는 무시하고 전체 풀로 폴백하는 편이 관대합니다.

## 사용법

```
pickme/index.html            # 전체 풀에서 추첨
pickme/index.html?theme=city # city 테마만 추첨 (city / space / tool / default)
```

- **카드 클릭** — 즉시 해당 스테이지로 이동 (셔플 중에도 가능)
- **■ STOP** — 자동 추첨 중단 → 수동 선택 모드
- **▶ RE-ROLL** — 중단 상태에서 다시 추첨
