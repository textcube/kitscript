# Eternal Invasion: Tank Edition

우주 배경의 실시간 진영 대결 캔버스 게임. 아군/적군 기지가 병사를 자동 생산하며 전진하고, 플레이어는 탱크를 조작(또는 오토파일럿)하여 적 기지를 파괴하면 승리하는 단일 파일(`index.html`) 웹 게임.

## 현재 구조 분석

### 기술 스택
- **단일 HTML 파일** (약 1000줄) — 외부 의존성 전무, 정적 호스팅 가능
- **렌더링**: Canvas 2D (1200×600 고정 해상도) + CSS 오버레이 HUD (글래스모피즘 패널, conic-gradient 쿨다운 링, 스캔라인 효과)
- **오디오**: Web Audio API 기반 자체 신스 엔진 (오실레이터 톤 — 발사/폭발/워프/피격음), 기본 OFF
- **상태 관리**: 전역 `state` 객체 + `CONFIG` 상수, 클래스 기반 엔티티 (`Unit` → `Base`/`Soldier`/`AITank`, `Projectile`, 이펙트 클래스들)
- **게임 루프**: `requestAnimationFrame` 직결 (update → draw)

### 주요 기능
| 기능 | 설명 |
|------|------|
| 자동 전투 | 양 진영 기지가 주기적으로 병사 생산, 병사는 자동 전진·교전 |
| 플레이어 탱크 | 오토파일럿(전선 유지 AI) 또는 WASD/화살표 수동 이동, 포탑은 자동 조준·발사 |
| 스킬 3종 | 1: 전술 폭격(STRK), 2: EMP 폭발, 3: 유도 미사일 살보(MSL) — 키보드/클릭 모두 가능 |
| 워프 귀환 | R/B 키로 기지 귀환 (3초 채널링), 기지 근처에서 HP/에너지 자동 회복 |
| 연출 | 파편(Shard) 파티클, 머즐 플래시, 사망 시퀀스, 플로팅 텍스트, 별/성운 패럴랙스 배경 |
| 종료 처리 | 기지 파괴 시 승패 오버레이 + 7초 자동 재배치 카운트다운 |

## 발견된 문제점

1. **프레임레이트 의존 게임 루프 (치명적)** — 모든 이동·쿨다운·타이머가 rAF 틱 단위라 144Hz 모니터에서는 게임 전체가 2.4배속, 60Hz 미만 저사양에서는 슬로모션이 됨. 델타타임 또는 고정 타임스텝이 없음.
2. **반응형 부재** — `#game-container`가 1200×600 고정. 작은 창/모바일에서 잘리고(`body overflow: hidden`이라 스크롤도 불가), 터치 조작 수단이 전혀 없어 모바일에서는 오토파일럿 관전만 가능.
3. **재시작이 `location.reload()`** — REBOOT 버튼, 게임 오버 자동 재시작 모두 전체 페이지 리로드. 오디오 ON 설정이 풀리고 로딩 화면부터 다시 시작. 상태 리셋 함수가 없음.
4. **잔버그**
   - 페이지 로드 완료 전 키 입력 시 `state.player`가 `null`이라 keydown 핸들러에서 예외
   - 탱크 사망 시 HP가 음수로 남아 HUD 바 width가 음수 %가 됨 (기지 HP는 클램프하면서 탱크 HP/에너지는 안 함)
   - 게임 종료 후에도 스킬의 `setTimeout` 폭발·미사일이 계속 발동 (오버레이 뒤에서 이펙트/사운드 재생)
   - 적 탱크 사망 후 부활이 `state.frame % 300 === 0` 조건이라 죽은 시점에 따라 부활 대기가 0~5초로 들쑥날쑥
   - `.circular-progress`의 `mask`(62/64%)와 `-webkit-mask`(58/60%) 수치 불일치로 브라우저별 링 두께가 다름
5. **게임성 부족** — 점수/킬 카운트/경과 시간 없음, 난이도 변화 없음(스폰 주기 고정이라 게임이 단조로운 소모전), EMP가 이름과 달리 적을 무력화하지 않고 단순 데미지만 줌, 궁극기 3종에 에너지 비용이 없어 에너지 자원의 의미가 약함(보조무기 10만 소모), 수동 조작 시에도 조준·발사가 전부 자동이라 "수동 모드"의 재미가 이동뿐.
6. **성능 낭비** — 매 프레임 병사마다 `filter().sort()`로 타겟 탐색(O(n² log n)), `shadowBlur` 남용(캔버스에서 가장 비싼 연산 중 하나), 파편 초과 시 `Array.shift()` 사용.
7. **일시정지 없음** — P키나 버튼으로 멈출 수 없음 (탭 전환 시 rAF 정지에만 의존).
8. **메타데이터 부재** — `<meta name="description">`, Open Graph 태그, 파비콘 없음. 사이트 내 다른 페이지(clinic, bizcard 등) 대비 SEO/공유 품질이 떨어짐.

## 개선 지시 사항 (→ Sonnet 5 작업)

> 아래 항목은 Sonnet 5 에이전트가 `index.html`을 직접 수정하여 구현한다. 단일 파일·의존성 없음 구조와 기존 비주얼 아이덴티티(네온 다크, BUNTGAMES 브랜딩 바)는 유지한다.

### 1. 고정 타임스텝 게임 루프 (최우선)
- `requestAnimationFrame` 콜백에서 경과 시간을 누적해 60 updates/sec 고정 타임스텝으로 `update()` 실행 (탭 복귀 시 폭주 방지를 위해 프레임당 최대 업데이트 수 제한)
- 기존 프레임 단위 상수(쿨다운, 스폰 주기, 워프 시간 등)는 60fps 기준이므로 값 변경 없이 그대로 동작해야 함

### 2. 반응형 스케일링 + 모바일 터치 조작 (최우선)
- 1200×600 내부 해상도는 유지하되, 뷰포트에 맞춰 `#game-container`를 `transform: scale()`로 축소 (letterbox 방식, resize 이벤트 대응)
- 모바일(터치 디바이스)에서 좌측 하단 가상 조이스틱으로 탱크 이동 (터치 시작 시 오토파일럿 자동 해제), 기존 스킬/워프 버튼은 터치 타깃 44px 이상으로 확대
- 모바일에서 HUD가 겹치지 않도록 배치 점검

### 3. 리로드 없는 재시작
- `resetGame()` 함수를 만들어 REBOOT 버튼과 게임 오버 자동 재시작이 페이지 리로드 없이 상태를 초기화하도록 변경 (오디오 ON/OFF, 오토파일럿 설정은 유지)
- 게임 종료 시 진행 중인 `setTimeout` 스킬 이펙트를 모두 취소하고, 오버레이를 닫고 즉시 재시작

### 4. 잔버그 수정
- keydown/클릭 핸들러에 `state.player` null/dead 가드 추가
- HP·에너지 HUD 바 width를 0~100%로 클램프
- 적 탱크 부활을 사망 시점 기준 고정 딜레이(예: 300프레임 카운트다운)로 변경
- `mask`/`-webkit-mask` 수치 통일

### 5. 게임성 강화
- **점수 시스템**: 킬 수·경과 시간 HUD 표시, 게임 오버 화면에 전투 결과(킬, 시간) 요약
- **난이도 상승**: 시간 경과에 따라 적 스폰 주기 단축(하한선 있음) 및 적 병사 능력치 소폭 강화 — 소모전 방지
- **EMP 리워크**: 범위 내 적 유닛을 3초간 기절(이동·발사 불가, 시각적 표시) + 기존 데미지 유지
- **에너지 비용**: 궁극기 1/2/3에 각각 에너지 비용 부여 (부족 시 발동 불가, HUD에 표시), 기지 회복의 전략적 가치 강화
- **수동 전투**: 수동 모드에서 마우스로 포탑 조준, 클릭으로 주포 발사 가능하게 (오토파일럿 시에는 기존 자동 조준 유지)

### 6. 성능 최적화
- 타겟 탐색을 `filter().sort()` 대신 단일 순회 최소거리 탐색으로 교체
- `shadowBlur` 사용을 핵심 오브젝트(탱크, 기지, 스킬 이펙트)로 축소, 파편·총알은 제거 또는 대폭 감소
- 파편 초과 제거를 `shift()` 대신 인덱스 순환 재사용(풀링)으로 변경

### 7. 편의 기능·마무리
- P키 + HUD 버튼으로 일시정지/재개 (일시정지 오버레이 표시)
- 첫 방문 시 조작법 안내(키보드/터치)를 로딩 후 짧게 표시하거나 시작 오버레이에 포함
- `<meta name="description">`, Open Graph 태그(title/description/type), 인라인 SVG 파비콘 추가

## 검증 체크리스트
- [ ] 60Hz/144Hz에서 게임 속도 동일
- [ ] 375px 모바일 뷰포트에서 게임 화면 전체가 보이고 터치로 이동·스킬 사용 가능
- [ ] REBOOT/게임 오버 재시작 시 페이지 리로드 없음, 오디오 설정 유지
- [ ] 로드 직후 키 연타 시 콘솔 에러 없음
- [ ] EMP 사용 시 적이 기절하고, 에너지 부족 시 궁극기 발동 불가
- [ ] 일시정지 중 모든 이동·타이머·사운드 정지
