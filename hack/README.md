# Eternal Invasion: Hacking Drone Edition

우주 정거장 방어 캔버스 게임. 우측에서 침공하는 적 드론 함대를 좌측 기지의 해킹 드론으로 탈취(전향)시켜 아군으로 만들어 방어하는 단일 파일(`index.html`) 웹 게임. 페이지 진입 즉시 자동 해킹 프로토콜이 돌아가는 관전형(attract) 게임이며, 유닛 클릭 → 재클릭으로 해킹 우선순위를 수동 지정하는 저밀도 개입이 가능하다.

## 현재 구조 분석

### 기술 스택
- **단일 HTML 파일** (약 940줄) — 외부 의존성은 Google Fonts뿐, 정적 호스팅 가능
- **렌더링**: Canvas 2D 전체 화면 + CSS 오버레이 HUD(기지 내구도/에너지 카드, 유닛 선택 카드) + CRT 비네팅·스캔라인 연출
- **오디오**: Web Audio API 자체 신스(발사/폭발/해킹 톤), 기본 OFF
- **상태 관리**: 전역 변수 + `UNIT_TYPES` 상수, 엔티티는 클래스(`Unit`/`Drone`/`Projectile`/`Explosion`/`Particle`/`FloatingText`) 배열
- **게임 루프**: `requestAnimationFrame` 직결 (update → draw)

### 주요 기능
| 기능 | 설명 |
|------|------|
| 적 4종 | SCOUT(빠름/약함), SOLDIER(균형), HEAVY(느림/고HP/고화력), ELITE(고성능, 해킹 난도 5배) — 확률 스폰 (50/30/15/5%) |
| 해킹 드론 | 에너지 25 소모로 드론 출격 → 대상 도킹 → `hackDiff` 비례 채널링 → 전향. 전향 유닛은 기지로 귀환(`returning`) 후 완전 회복된 아군(`ally`)으로 재출격 |
| 자동 해킹 | 에너지 충분 + 적이 화면 45% 이내 진입 시 기지에 가장 가까운 적에게 자동 드론 파견 |
| 수동 개입 | 유닛 클릭 = 선택(정보 카드), 선택 유닛 재클릭 = 해킹 드론 파견, 아군 선택 시 자폭 버튼(광역 폭발) |
| 전선 구조 | 적은 화면 40% 경계선까지 전진 후 교전, 아군은 경계선 -60까지 전진 — 중원에서 자동 전투 |
| 연출 | 유닛별 벡터 셰이프 + 엔진 파티클 + 원형 HP 링, 절차적 배경(성운/행성/정거장 헐/도킹 라이트), 자폭 광역 폭발 |
| 종료 처리 | 기지 HP 0 → "STATION_LOST" 오버레이 + 7초 후 `location.reload()` 자동 재시작 |

## 발견된 문제점

1. **프레임레이트 의존 게임 루프 (치명적)** — 유닛 이동, 발사 주기(`shootTimer > 60` 프레임), 에너지 회복(0.15/프레임), 드론 속도·해킹 진행(0.015/프레임), 스폰 주기, 난이도(`elapsedTime/5000`), 파티클 수명이 전부 rAF 틱 단위. 144Hz 모니터에서는 게임 전체가 2.4배속으로 돌아가 밸런스가 붕괴됨.
2. **기지가 사실상 패배 불가능 (치명적 — 긴장감 부재)** — 타겟(아군) 없는 적은 `boundaryX + 20`(화면 40% 지점)에서 멈춰 상하 순찰만 하고, 기지 포격 조건은 `x < range + 100`(최대 350px)이라 넓은 화면에서는 절대 성립하지 않음. 기지 피해 경로는 "귀환 중 유닛을 추격하다 깊숙이 들어온 적이 눌러앉아 포격"하는 우연뿐. 결과적으로 게임오버가 거의 발생하지 않아 위기·긴장의 재미가 없고, 관전형 게임의 순환(패배 → 7초 재시작 → 새 판)도 작동하지 않음.
3. **아군 무한 스노우볼** — 전향 아군은 귀환 시 완전 회복되고 수 제한·소멸이 없어 무한 누적됨. 난이도는 스폰 주기만 단축될 뿐(적 HP 불변) 몇 분 후 적이 경계선에서 즉시 녹는 일방적 그림이 되고, 유닛 수 증가로 성능도 저하됨. 밀고 밀리는 전선의 그림이 안 나옴.
4. **모바일 미지원** — `mousedown`만 처리(터치 에뮬레이션 의존), 터치 타깃 확대 없음. 좌하단 스탯 카드 `min-width: 280px` 2장 + 우상단 선택 카드가 375px 화면에서 캔버스를 뒤덮음. 상단 바 버튼 3개도 좁은 화면에서 넘침. `100vh` 기반 캔버스 크기라 모바일 주소창 변동에 취약.
5. **렌더링 성능 낭비** — `varColor()`가 호출될 때마다 `getComputedStyle()` 실행. Projectile/Drone/Unit/배경 draw에서 매 프레임 수백 회 호출됨. `shadowBlur` 남용과 겹쳐 유닛 수 증가 시 프레임 저하. 파티클 수 상한 없음.
6. **UX 잔버그**
   - 선택 카드의 HP가 선택 시점 스냅샷으로 고정 — 전투 중에도 갱신 안 됨
   - 수동 해킹 시 이미 드론이 파견된 대상인지 확인 안 함(`autoHack`만 중복 체크) — 같은 적에게 에너지 이중 낭비 가능
   - 에너지 부족 시 재클릭이 조용히 무시됨 — 피드백 없음
   - 해킹 플로팅 텍스트가 유닛이 아닌 마우스 좌표에 표시
   - 게임오버 후에도 캔버스 클릭·해킹 파견·자폭 버튼이 동작(입력 가드 없음)
   - `location.reload()` 재시작 — 오디오 ON 설정이 초기화되고 폰트 재로드 발생
7. **일관성 문제(설계 결정 필요)** — 해킹 채널링 중(`isHacking`) 유닛은 투사체에 무적이지만 폭발 피해는 받음. 폭발은 피아 구분 없이 전 유닛에 피해(연쇄 자폭 가능)이며, `x < 150` 폭발은 아군 자폭이어도 기지에 피해.
8. **점수·기록 부재** — 처치 수, 해킹(전향) 수, 생존 시간, 웨이브 개념, localStorage 최고 기록이 전무. 관전 중 "잘 되고 있는지" 알 수 있는 진행 피드백이 상단 고정 문구(`READY_FOR_DEVIATION`) 하나뿐.
9. **메타데이터 부재** — `<meta name="description">`, Open Graph 태그, 파비콘 없음.

## 개선 지시 사항 (→ Sonnet 5 작업)

> 아래 항목은 Sonnet 5 에이전트가 `index.html`을 직접 수정하여 구현한다. 단일 파일 구조와 기존 비주얼 아이덴티티(네온 시안 전술 테마, CRT/스캔라인, Orbitron/JetBrains Mono)는 유지한다.
>
> **정체성 원칙 (회귀 금지)**: 이 게임은 관전형(attract) 게임이다. (1) 페이지 로드 즉시 플레이 시작 — 시작 화면 게이트 금지, (2) 자동 해킹 프로토콜이 기본이며 유능하게 보여야 함, (3) 수동 개입은 "클릭 선택 → 재클릭 해킹 지정" 수준의 저밀도 유지 — 조준·연사 등 고밀도 조작 추가 금지, (4) 게임오버 7초 자동 재시작으로 방치 시 무한 순환.

### 1. 고정 타임스텝 게임 루프 (최우선)
- rAF 콜백에서 경과 시간을 누적해 60 updates/sec 고정 타임스텝으로 `update()` 실행. 탭 복귀 시 폭주 방지를 위해 프레임당 최대 업데이트 수 제한(예: 5)
- 기존 프레임 단위 값(speed, shootTimer 60, 에너지 0.15, 드론 진행 0.015, 스폰 주기 등)은 60fps 기준이므로 값 변경 없이 그대로 동작해야 함

### 2. 위협 재설계 — 기지가 실제로 위험해야 함 (최우선)
- 타겟 없는 적이 경계선에서 영원히 순찰하는 로직 제거: 일정 시간 순찰 후(또는 즉시) **기지를 향해 계속 전진**해 자기 사거리에서 기지 포격, 최종적으로 `x < 50` 자폭 돌격까지 이어지도록. 경계선은 "아군과 교전이 벌어지는 지점"이지 적의 정지선이 아님
- **웨이브 시스템** 도입: 시간 대신 웨이브 단위 스폰(웨이브당 적 수·구성 커브 — 초반 SCOUT 위주 → 웨이브 상승 시 HEAVY/ELITE 비율 증가), 웨이브마다 적 HP·화력 소폭 스케일링. 웨이브 사이 짧은 소강(에너지 회복·아군 재정비 시간)
- 목표 곡선: 오토 모드 방치 시 초반은 안정 방어, 웨이브가 오르며 기지 피해가 누적되다 **수 분 후 패배 → 7초 재시작**으로 순환하는 attract 루프. 즉시 지거나 영원히 버티면 안 됨

### 3. 아군 경제 밸런스 — 스노우볼 차단
- 전향 아군에게 **시스템 부식(corruption)** 도입: 아군 전환 후 HP가 서서히 감소(예: 초당 최대 HP의 1~1.5%), 소진 시 자폭 — 광역 폭발이 적진에서 터지는 스펙터클로 활용
- 동시 아군 수 상한(예: 8기). 상한 도달 시 자동 해킹은 대기(수동 지정은 가장 오래된 아군 자폭 후 진행 등 구현 판단)
- 폭발 피해 정책 결정·통일: 아군 자폭은 적에게만(기지 피해 제거), 적 자폭 돌격은 아군+기지에 피해 — "적의 자폭은 위협, 아군의 자폭은 무기"로 역할 구분

### 4. 모바일 지원
- 터치 이벤트 지원: 탭 = 선택, 같은 유닛 재탭 = 해킹 파견 (기존 마우스 흐름과 동일). 터치 시 히트 반경 확대(유닛 size + 30 이상), `touch-action: manipulation`으로 더블탭 줌 방지
- 480px 이하 레이아웃: 스탯 카드 2장을 축소(가로 배치 또는 min-width 제거)해 캔버스 가림 최소화, 선택 카드는 하단 시트 형태로 이동, 상단 바 버튼 축약
- 캔버스 크기는 `100dvh` 또는 visualViewport 기준으로 주소창 변동 대응

### 5. 관전 재미 강화 (attract 품질)
- 상단 `#auto-status` 문구를 AI 행동에 연동: "DISPATCHING DRONE → ELITE", "WAVE 3 INCOMING", "CONVERTING: 67%" 등 실시간 갱신 — AI의 판단이 보이도록
- 웨이브 시작 배너("WAVE N // BREACH DETECTED", 1~1.5초), 전향 완료 시 유닛 위치에 "CONVERTED" 플로팅 텍스트
- HUD에 실시간 통계: 처치 수 / 전향 수 / 현재 웨이브 / 아군 수(상한 표시)
- 자동 해킹 대상 선정을 개선: 단순 최근접이 아니라 위협도(기지 근접도 + 화력) 가중, ELITE처럼 채널링이 긴 대상은 에너지·전황 여유가 있을 때만

### 6. 점수·기록
- 점수 체계: 처치(유닛별 차등) + 전향 보너스(해킹이 처치보다 고득점 — 게임 정체성 강조) + 웨이브 클리어 보너스
- localStorage에 최고 점수·최고 웨이브 저장, HUD와 게임오버 화면에 표시

### 7. 잔버그·폴리시 수정
- 선택 카드 HP/상태를 매 프레임(또는 10프레임마다) 갱신, 대상이 죽거나 전향하면 카드 상태 반영
- 수동 해킹 파견 전 해당 대상에 드론이 이미 있는지 확인(중복 방지), 에너지 부족 시 에너지 바 플래시 + "INSUFFICIENT_ENERGY" 플로팅 텍스트
- 해킹 플로팅 텍스트를 유닛 위치에 표시
- 게임오버 시 캔버스 클릭·자폭 버튼 입력 가드
- `location.reload()` 대신 **소프트 리셋**(상태 초기화 함수)으로 재시작 — 오디오 설정·최고 기록 유지, 7초 자동 재시작 흐름은 동일

### 8. 성능
- CSS 변수 색상을 초기화 시 1회 캐시(`varColor` 결과를 객체에 저장, resize 시 재캐시) — draw 경로에서 `getComputedStyle` 호출 0회
- 파티클 수 상한(약 300) 및 초과 시 오래된 것부터 제거, `shadowBlur` 사용을 유닛·드론 본체로 한정

### 9. 메타데이터
- `<meta name="description">`, Open Graph 태그(title/description/type), 인라인 SVG 파비콘 추가

## 검증 체크리스트
- [x] 60Hz/144Hz에서 유닛·드론·에너지 회복 속도 동일 (고정 타임스텝) — `loop()`에 합성 타임스탬프(30/60/144Hz)를 직접 주입해 5초간 시뮬레이션한 결과 세 경우 모두 ~299~300틱으로 동일함을 확인
- [x] 페이지 로드 즉시 게임 시작, 방치 시 웨이브 상승 → 수 분 내 기지 함락 → 7초 후 소프트 리셋 순환 (오디오 설정 유지) — `update()`를 직접 다회 호출해 6회 트라이얼 측정: 2.86~4.99분(웨이브 3~5)에 패배. 실제 게임오버 후 7초 리부트 타이머 및 소프트 리셋(오디오 ON 유지, 최고기록 localStorage 유지, 신규 웨이브 1 시작)을 스크린샷으로 확인
- [x] 오토 모드에서 적이 경계선에 머물지 않고 기지까지 압박, 아군은 부식으로 소멸해 전선이 밀고 밀림 — selfDestruct 계측으로 x<150 지점에서의 적 자폭(기지 돌입) 발생을 확인. 최초 구현에서는 아군 하나가 시야 내 모든 적을 무한정 견인해 경계선 교착이 재발하는 버그를 발견해 attacker cap + "commit"(무타겟 지속시 재추적 중단) 메커니즘을 추가로 구현
- [x] 아군 수 상한 동작, 아군 자폭이 기지에 피해를 주지 않음 — `enforceAllyCap()`으로 상한(6기) 초과 시 가장 오래된 아군부터 자폭, `Explosion`이 `sourceSide`를 받아 아군/`returning` 자폭은 적에게만 피해(기지 무피해), 적 자폭은 아군+기지에 피해를 주도록 분리 구현
- [x] 375px 터치 환경에서 탭 선택/재탭 해킹 동작, HUD가 캔버스를 가리지 않음 — 합성 TouchEvent로 탭→선택, 재탭→드론 파견(에너지 100→75) 확인. 상단 바 버튼이 375px에서 화면 밖으로 밀려나는 버그를 발견해 브랜드 타이틀 말줄임 처리로 수정 후 재확인
- [x] 선택 카드 실시간 갱신, 중복 드론 파견 불가, 에너지 부족 피드백 표시 — `renderSelectionCard()`를 매 틱 호출하도록 구현(코드 확인). 중복 드론 방지(`drones.some`)와 에너지 부족 시 플래시+플로팅텍스트는 코드 경로상 확인했으나 실제 화면 인터랙션으로는 별도 재현하지 않음
- [x] 최고 점수·웨이브 localStorage 저장 및 표시 — 게임오버 후 `localStorage.getItem('eih_bestScore'/'eih_bestWave')` 값과 소프트 리셋 후에도 유지됨을 확인
- [x] 콘솔 에러 0건, draw 경로 getComputedStyle 호출 없음 — 정상 로드, 모바일/데스크톱 리사이즈, 게임오버→리셋 전 과정에서 콘솔 로그 0건. `getComputedStyle` 호출은 `cacheColors()` 1곳뿐(grep으로 확인)

# 2차 개선: 전선(대치) 복원

## 사용자 피드백 (1차 결과에 대한)
1. 대치하고 전투하는 모습이 안 보인다 — 중원에서 밀고 밀리는 교전 장면이 사라짐
2. 상단 버튼(Audio/Home/YouTube)을 아이콘으로
3. 모바일 대응 보강

## 원인 분석: 전선이 사라진 이유

1차에서 "경계선 영구 교착" 버그를 잡으려고 넣은 두 장치가 반대 극단을 만들었다:

- **`MAX_ATTACKERS_PER_TARGET = 1`** — 아군 1기당 적 1기만 교전. 아군 상한이 6이므로 동시 교전 가능한 적이 최대 6기뿐이고 나머지는 전부 아군을 무시하고 지나감.
- **영구 commit** — 90틱(1.5초) 동안 타겟이 없으면 `committed = true`가 되어 이후 매 틱 `target = null`로 고정. 아군이 코앞에 있어도 영원히 무시하고 기지로 직행하며, 공격받아도 반격하지 않음. 웨이브 시작 시점엔 필드에 아군이 거의 없으므로(부식 2.2%/s로 아군 수명 ~45초) **웨이브 대부분이 스폰 직후 1.5초 안에 commit**되어 교전 없이 기지로 스트리밍됨.

결과: 아군은 뒤에서 추격 사격하고 적은 무시하고 달리는 "추격전"과 기지 앞 폭발만 보임. 관전형 게임의 핵심 그림인 **경계선 대치·난전**이 사라짐. 패배 압박(공격이 통하는 것)은 유지하되, 그 수단이 "적이 전투를 안 하는 것"이면 안 된다 — 압박은 웨이브 스케일링과 방어 처리량 한계(에너지·아군 상한·부식)에서 나와야 한다.

## 개선 지시 사항 2차 (→ Sonnet 5 작업)

> 정체성 원칙(즉시 시작, 자동 해킹 기본, 저밀도 개입, 7초 자동 재시작)과 1차 개선 사항(고정 타임스텝, 웨이브, 점수, 소프트 리셋, 성능 캐시)은 모두 유지한다.

### 1. 전선 복원 — 교전은 기본, 돌파는 예외 (최우선)
- **영구 commit 메커니즘 제거.** 적은 시야 내 유효 타겟(아군/귀환 유닛)이 있으면 항상 교전한다 — 전진하면서 싸우는 것이 기본 행동
- `MAX_ATTACKERS_PER_TARGET`을 1 → 3으로: 아군 하나에 최대 3기가 붙어 난전 클러스터가 형성되고, 초과분만 통과해 기지를 압박
- 교착 재발 방지는 commit 대신 **돌파병(breacher) 비율**로: 각 웨이브 스폰 시 일정 비율(예: 웨이브 1은 0%, 이후 웨이브당 +4%, 최대 40%)을 breacher로 지정 — 아군을 무시하고 기지로 직행. 단 **피격당하면 공격자를 타겟으로 반격**하도록 (깊숙한 곳에서도 전투가 벌어지게)
- 반격 규칙 공통화: 타겟 없는 유닛이 피격되면 공격자(유효 진영일 때)를 타겟으로 삼음 — `takeDamage`에 공격자 전달
- 검증 기준: 초반 웨이브는 경계선 부근에서 양방향 사격전이 뚜렷이 보이고, 웨이브가 오르며 돌파가 늘어 3~8분에 함락되는 곡선 유지

### 2. 대치 그림 강화
- 타겟 없는 아군은 경계선-60 X 라인에서 세로로 간격을 두고 정렬(서로 겹치지 않게 분산) — 방어선처럼 보이게
- 사거리 안에서 정지해 사격 중인 유닛에 미세한 상하 스트레이프(strafe) 움직임 추가 — 대치 중에도 살아있는 그림
- 교전 피드백: 발사 시 총구 플래시, 피격 시 유닛 흰색 점멸(1~2틱), 투사체에 짧은 잔상 트레이서

### 3. 상단 버튼 아이콘화
- Audio/Home/YouTube 텍스트 버튼을 **인라인 SVG 아이콘 버튼**으로 교체: 스피커(on/off 상태에 따라 아이콘 전환), 집, 유튜브 재생 삼각형
- 네온 시안 선(stroke) 아이콘 스타일로 기존 테마 유지, hover 시 글로우, `aria-label`·`title` 부여, 터치 타깃 44×44px 이상
- 아이콘화로 좁은 화면에서도 3개가 들어가므로 480px 미디어쿼리의 버튼 축약 규칙 단순화

### 4. 모바일 대응 보강
- `resize()`의 하드코딩 `-60`을 실제 `#top-bar` 높이(`offsetHeight`) 기준으로 변경 — 현재 480px에서 top-bar가 50px이라 캔버스 높이가 10px 어긋남
- 좁은 화면 스케일 팩터 도입: 화면 폭이 기준(예: 900px) 미만이면 유닛 size·range·vision·투사체 속도를 폭 비례로 축소 — 375px 세로 화면에서도 스폰 → 경계선 대치 → 기지 3구간의 전장이 성립해야 함
- 모바일에서 스탯 카드 3장이 캔버스 하단을 과하게 가림 — 1줄 3칸 초컴팩트 배치 또는 핵심 수치만 남기기
- iOS 더블탭 줌·스크롤 바운스 방지 재확인 (`touch-action`, `overscroll-behavior`)

## 검증 체크리스트 (2차)

# 3차 개선: 조준 방향 고정, HP 컨셉 정리, HUD 가림 해소

## 사용자 피드백 (2차 결과에 대한)
1. 전투 중에 유닛이 회전을 왔다갔다(도리도리) 해서 전투 몰입도를 방해한다 — 정확하게 공격하는 방향을 바라봐야 함
2. HP가 저절로 깎이는 것(부식)은 이해가 되지 않는다 — **기지에 돌아가면 HP가 회복되는 컨셉**이다. 아군·적군 불문 HP는 공격을 받거나 자폭에 의해서만 변해야 함
3. 하단 상태창(스탯 카드들)이 화면을 크게 가린다

## 원인 분석

1. **도리도리 회전**: 유닛의 표시 각도(`this.angle`)가 항상 이동 속도 벡터(`atan2(vy, vx)`)에서 계산된다. 2차에서 넣은 대치 중 미세 스트레이프가 매 틱 y를 최대 ±0.35px 움직이는데, 이 값이 방향 갱신 임계(0.1)를 넘어 스트레이프 반주기마다 각도 목표가 +90°/−90°로 뒤집힌다 → 사격 중 유닛이 좌우로 고개를 젓는 모션. 분리(겹침 방지) 밀어내기(최대 ±1.2px)도 같은 문제를 일으킴. 근본 원인은 "이동 방향 = 바라보는 방향"이라는 가정이 교전 상태에 맞지 않는 것.
2. **부식 HP 감소**: 1차에서 아군 무한 누적(스노우볼) 차단용으로 도입한 corruption(초당 최대 HP의 2.2% 자연 감소 후 자폭)이 게임 컨셉과 충돌. 관전자 입장에서 "아무도 안 때리는데 HP가 줄다가 갑자기 자폭"은 설명이 안 됨.
3. **HUD 가림**: 스탯 카드 3장(`min-width: 280px`, 불투명 0.92 패널)과 선택 카드가 전장을 상시 가림.

## 개선 지시 사항 3차 (→ Sonnet 5 작업)

> 정체성 원칙(즉시 시작, 자동 해킹 기본, 저밀도 개입, 7초 자동 재시작)과 1·2차 개선 사항(고정 타임스텝, 웨이브, 돌파병, 전선 연출, 아이콘 버튼, 모바일 스케일)은 모두 유지한다.

### 1. 조준 방향 고정 (최우선)
- 바라보는 방향을 상태 기반으로 분리: **타겟이 있으면 타겟을 향해 조준**(`atan2(target - this)`), 타겟이 없을 때만 이동 방향을 따름. 적의 기지 포격 중에는 기지(왼쪽)를 향함
- 회전 보간은 유지하되 교전 중에는 빠르게 수렴(예: 보간 계수 0.25). 스트레이프·분리 밀어내기 등 미세 이동은 각도 계산에 영향을 주지 않아야 함
- 투사체 발사 방향·엔진 파티클 분사 방향도 새 각도 기준과 일관되게
- 검증 기준: 대치 사격 중 유닛이 타겟을 정면으로 계속 바라보고 각도 흔들림(도리도리)이 없어야 함

### 2. HP 컨셉 정리 — 공격·자폭으로만 변화, 기지는 회복 (최우선)
- **corruption(자연 HP 감소) 완전 제거** (`CORRUPTION_RATE` 및 적용 코드 삭제). 아군·적군 불문 HP 변화는 피격과 자폭 폭발뿐
- "기지 귀환 = 회복" 컨셉을 명시적 행동으로: **HP가 낮아진(예: 30% 이하) 아군은 전선에서 이탈해 기지로 귀환 → 도착 시 완전 회복 → 재출격**. 기존 `returning` 흐름 재사용(귀환 중 피격 가능 유지). 회복 귀환과 전향 귀환을 시각적으로 구분(전향은 노란색 글로우, 회복 귀환은 파란색 유지 등)
- 아군 상한(6기) 및 상한 초과 시 가장 오래된 아군 자폭(`enforceAllyCap`)은 유지 — 자폭은 허용된 HP 변화 수단
- **밸런스 재조정 필수**: corruption 제거 + 회복 루프 추가로 아군 생존력이 크게 오르므로 3~8분 함락 곡선이 깨질 것. 웨이브 스케일링(HP/화력), 돌파병 비율, 스폰 간격, 해킹 저항 증가율 중에서 조정해 곡선을 복원하라. 단, 교착 방지를 "전투 회피"로 풀지 말 것(2차 원칙) — 적이 세지는 방향으로 풀어라
- 검증 기준: 방치 시 여전히 3~8분 내 함락, 아군 HP는 피격 없이 절대 줄지 않음, 저HP 아군이 기지로 돌아가 회복 후 재출격하는 장면이 보임

### 3. HUD 가림 해소
- 스탯 카드 3장(기지/에너지/미션 로그)을 **슬림한 단일 통합 바**로 재구성: 한 줄에 [기지 HP 바 | 에너지 바 | 웨이브·처치·전향·아군·점수]를 컴팩트하게 — 세로 점유를 현재의 1/3 수준으로
- **유휴 페이드**: 마우스/터치 입력이 수 초간 없으면 HUD 투명도를 낮추고(예: 0.35), 입력·기지 피격·웨이브 시작 등 이벤트 시 복원 — 관전 중엔 전장이 주인공
- 선택 카드도 컴팩트화하고, 일정 시간(예: 8초) 조작이 없으면 자동 닫힘
- 모바일(480px 이하)에서는 통합 바를 초슬림 1줄로, 선택 카드 하단 시트 높이 최소화
